## 一、代码 Bug（必须修复，否则跑不起来）

### Bug 1：`src/routes/agent.ts` 第 12-15 行有错误的 import

```
12  import chatRouter from './routes/chat.js'
13  import knowledgeRouter from './routes/knowledge.js'
14  import analyzeRouter from './routes/analyze.js'
15  import configRouter from './routes/config.js'
```

这 4 行不应该出现在 `agent.ts` 里。这些是 `index.ts` 的 import，你复制粘贴时带进来了。`agent.ts` 自身是 `src/routes/` 目录下的文件，路径 `./routes/chat.js` 不存在，**会直接报 import 错误导致启动失败**。

**操作**：删掉第 12-15 行。

---

### Bug 2：`src/agent/nodes/classifyNode.ts` 引用了不存在的 `state.hasFile`

第 33 行：
```
33  const fullContext = state.hasFile && state.documentText
```

你的 `state.ts` 里没有 `hasFile` 字段。这会导致 TypeScript 编译报错。

**操作**：二选一：
- **方案 A**（推荐）：改第 33 行为 `const fullContext = state.documentText`，去掉 `hasFile` 判断，直接用 `documentText` 是否为空来判断
- **方案 B**：在 `state.ts` 第 15 行后加 `hasFile: Annotation<boolean>({ reducer: (_, x) => x, default: () => false })`

---

### Bug 3：`src/agent/nodes/classifyNode.ts` 和 `mainGraph.ts` 里有重复的 classifyNode

`mainGraph.ts` 第 18-64 行定义了完整的 `classifyNode` 函数，`nodes/classifyNode.ts` 也定义了一个 `classifyNode` 函数。但 `mainGraph.ts` 用的是**自己内部定义的那个**，根本没有 import `nodes/classifyNode.ts`。

两个 `classifyNode` 实现还不一样：
- `mainGraph.ts` 第 39-52 行有完整的 prompt（分类与校验规则 1234），和你 demo 一致
- `nodes/classifyNode.ts` 第 37 行用 `classifierModel.invoke(fullContext)` 直接传字符串，没有包装成 `HumanMessage`，且没有 prompt 规则

**操作**：二选一：
- **方案 A**（推荐，改动最小）：`nodes/classifyNode.ts` 整个文件删掉，因为 `mainGraph.ts` 已经包含了完整的 classifyNode
- **方案 B**：让 `mainGraph.ts` 删掉内部的 classifyNode，改为 `import { classifyNode } from './nodes/classifyNode.js'`，但需要先修好 `classifyNode.ts` 的 prompt 和 invoke 方式

---

### Bug 4：`src/agent/nodes/clarifyNode.ts` 没有 export

第 7 行：
```
7  async function askForMoreNode(state: MainStateType): Promise<Partial<MainStateType>> {
```

函数没有 `export`。但 `mainGraph.ts` 也没有 import 它——`mainGraph.ts` 第 68-79 行有自己内部的 `askForMoreNode`。

和 Bug 3 同样的问题：两个地方写了重复的函数。

**操作**：`nodes/clarifyNode.ts` 整个文件删掉，`mainGraph.ts` 已经包含了完整的 askForMoreNode。

---

### Bug 5：`src/agent/subGraphs/consultGraph.ts` 第 4 行缺少 `.js` 后缀

```
4  import { ConsultState, ConsultStateType } from '../state'
```

项目是 ESM（`"type": "module"`），TypeScript 的 ESM 模式要求 import 路径带 `.js` 后缀。

**操作**：改为 `import { ConsultState, ConsultStateType } from '../state.js'`

---

### Bug 6：`src/agent/subGraphs/applyGraph.ts` 第 4 行同样缺少 `.js` 后缀

```
4  import { ApplyState, ApplyStateType } from '../state'
```

**操作**：改为 `import { ApplyState, ApplyStateType } from '../state.js'`

---

### Bug 7：`src/agent/subGraphs/applyGraph.ts` 第 93-96 行和第 103 行的动态 import 写法有问题

```typescript
94  return { messages: [new (await import('@langchain/core/messages')).AIMessage(
95    '根据您提供的材料...'
96  )] }
...
103 const { AIMessage } = await import('@langchain/core/messages')
```

在文件顶部第 6 行已经 import 了 `HumanMessage`，但漏了 `AIMessage`。不需要用动态 `await import`，直接在第 6 行加上就行。

**操作**：
- 第 6 行改为 `import { HumanMessage, SystemMessage, AIMessage } from '@langchain/core/messages'`
- 第 94 行改为 `return { messages: [new AIMessage('根据您提供的材料...')] }`
- 第 103 行删掉

---

### Bug 8：`package.json` 缺少 `zod` 依赖

新代码里大量使用了 `import { z } from 'zod'`（`classifyNode`、`applyGraph` 等），但 `package.json` 没有 `zod`。

**操作**：执行 `pnpm add zod`

```json
{
  "compilerOptions": {
    "target": "ES2022",
    "module": "ESNext",
    "moduleResolution": "node",
    "outDir": "./dist",
    "rootDir": "./src",
    "strict": true,
```

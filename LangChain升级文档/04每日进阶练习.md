## 先回答三个关键问题

### Q1：RAG 需要换"真实数据库"吗？

**不需要。** 你项目里的 `vectorStore.ts` 用 SQLite + 手写余弦相似度，已经是一个完整可用的向量数据库。你只需要把它**封装成 LangGraph 节点**，而不是换成 pgvector/Chroma。

如果将来数据量超过 10 万条才需要考虑迁移，现阶段 SQLite 完全够用。

### Q2：为什么不用 LCEL，要用 LangGraph？

| 维度 | LCEL Chain（`pipe`） | LangGraph StateGraph |
|------|---------------------|----------------------|
| 流程形状 | 线性，A→B→C | 有向图，支持分支、循环、回退 |
| 条件路由 | 做不到 | 原生 `addConditionalEdges` |
| 暂停等待 | 做不到 | `interrupt()` 暂停等人工输入 |
| 并行执行 | 做不到 | `Send` API 同时跑多个节点 |
| 状态持久化 | 做不到 | `checkpointer` 随时恢复 |

你想要的"意图分类 → 三路分流 → 信息不足时反问"——这个结构只有 LangGraph 能做，LCEL 做不了。

### Q3：还需要学什么才能完成目标？

你已经学完了 Day 1–15，掌握了 StateGraph 基础（条件边、节点）。

**还差的核心概念：**
1. **`MessagesAnnotation` + 自定义复合 State**（Day 16–17）：让节点之间优雅传递消息和业务数据
2. **`interrupt()` + `Command`**（Day 18）：实现"信息不足 → 反问 → 继续"的兜底流程
3. **`Send` 并行节点**（Day 19）：同时执行互不依赖的步骤
4. **子图（Subgraph）**（Day 20）：把三条路径各自封装为独立子图，主图只做分发
5. **与你现有服务集成**（Day 22–26）：把 `vectorStore.ts`、`docParser.ts` 接入 LangGraph 节点

## 第三阶段：LangGraph 进阶（Day 16–21）

### Day 16 — MessagesAnnotation（消息原生状态）

**今天学什么**：Day 15 的 `StateGraph` 用的是自定义 `channels`，手动管理消息很繁琐。`MessagesAnnotation` 是 LangGraph 内置的消息状态，自动处理消息追加（append），让节点可以像对话一样叠加消息。

**做什么**：用 `MessagesAnnotation` 重写 Day 15 的意图分流示例，对比代码量和可读性。

```typescript
// day-16.ts
import 'dotenv/config'
import { StateGraph, MessagesAnnotation, START, END } from '@langchain/langgraph'
import { ChatOpenAI } from '@langchain/openai'
import { HumanMessage, AIMessage, SystemMessage } from '@langchain/core/messages'
import { z } from 'zod'

const model = new ChatOpenAI({
  apiKey: process.env.QWEN_API_KEY!,
  configuration: { baseURL: process.env.QWEN_BASE_URL },
  modelName: process.env.QWEN_MODEL,
  temperature: 0,
})

// 意图分类模型（结构化输出）
const intentModel = model.withStructuredOutput(
  z.object({ intent: z.enum(['consult', 'apply', 'other']) })
)

// 节点1：意图分类（只读消息，不添加新消息）
async function classifyNode(state: typeof MessagesAnnotation.State) {
  const lastMsg = state.messages.at(-1)!
  const result = await intentModel.invoke([
    new SystemMessage('判断用户意图：consult=咨询问题，apply=申请表单，other=其他'),
    new HumanMessage(String(lastMsg.content)),
  ])
  // 关键：返回空对象不修改 messages，只做路由决策
  console.log('  → 意图:', result.intent)
  // 把意图记在消息 metadata 里（临时用法，Day17 会用更好的方式）
  return { messages: [new AIMessage({ content: '', additional_kwargs: { intent: result.intent } })] }
}

// 节点2a：咨询回复
async function consultNode(state: typeof MessagesAnnotation.State) {
  const userMsg = state.messages.find(m => m._getType() === 'human')!
  const reply = await model.invoke([
    new SystemMessage('你是政策咨询助手，简洁回答用户的咨询问题。'),
    new HumanMessage(String(userMsg.content)),
  ])
  return { messages: [reply] }  // 追加 AIMessage 到消息列表
}

// 节点2b：申请引导
async function applyNode(state: typeof MessagesAnnotation.State) {
  const userMsg = state.messages.find(m => m._getType() === 'human')!
  const reply = await model.invoke([
    new SystemMessage('你是申请表单助手，引导用户填写申请材料，列出需要准备的信息。'),
    new HumanMessage(String(userMsg.content)),
  ])
  return { messages: [reply] }
}

// 节点2c：兜底
async function otherNode(state: typeof MessagesAnnotation.State) {
  return {
    messages: [new AIMessage('您的问题我暂时无法处理，请描述您是要"咨询政策"还是"申请加分"？')]
  }
}

// 路由函数：读取最后一条 AI 消息里存的 intent
function routeIntent(state: typeof MessagesAnnotation.State) {
  const aiMsgs = state.messages.filter(m => m._getType() === 'ai')
  const lastAi = aiMsgs.at(-1)
  const intent = lastAi?.additional_kwargs?.intent as string | undefined
  return intent ?? 'other'
}

// 构建图（MessagesAnnotation 作为状态，注意写法）
const graph = new StateGraph(MessagesAnnotation)
  .addNode('classify', classifyNode)
  .addNode('consult', consultNode)
  .addNode('apply', applyNode)
  .addNode('other', otherNode)
  .addEdge(START, 'classify')
  .addConditionalEdges('classify', routeIntent, {
    consult: 'consult',
    apply: 'apply',
    other: 'other',
  })
  .addEdge('consult', END)
  .addEdge('apply', END)
  .addEdge('other', END)

const app = graph.compile()

const inputs = [
  '推免加分里，挑战杯国家级一等奖能加多少分？',
  '我想申请科研成果加分，需要准备什么材料？',
  '明天天气怎么样？',
]

for (const input of inputs) {
  console.log(`\n输入: ${input}`)
  const result = await app.invoke({ messages: [new HumanMessage(input)] })
  const lastAI = result.messages.filter((m: { _getType: () => string }) => m._getType() === 'ai').at(-1)
  console.log(`回复: ${String(lastAI?.content ?? '（无内容）').slice(0, 100)}`)
}
```

**验收**：三种输入分别走到不同节点，推理链中能看到 classify → consult/apply/other 的路由。

**关键理解**：`MessagesAnnotation` 的 reducer 是 **append**（追加），不是替换。每次节点 return `{ messages: [新消息] }` 都是往列表末尾追加，这与 Day 15 的 `reducer: (_, x) => x`（替换）完全不同。

### Day 17 — 自定义复合 State（Annotation.Root）

**今天学什么**：实际项目里需要在消息之外存储业务数据（如意图、文档内容、置信度）。`Annotation.Root` 允许把 `MessagesAnnotation` 和自定义字段组合成一个复合状态。

**做什么**：把 Day 16 的意图改为存在独立的状态字段里，而不是塞进消息的 `additional_kwargs`。

```typescript
// day-17.ts
import 'dotenv/config'
import { StateGraph, MessagesAnnotation, Annotation, START, END } from '@langchain/langgraph'
import { ChatOpenAI } from '@langchain/openai'
import { HumanMessage, AIMessage, SystemMessage } from '@langchain/core/messages'
import { z } from 'zod'

// ✅ 复合状态：消息 + 业务字段
const AppState = Annotation.Root({
  ...MessagesAnnotation.spec,                       // 包含 messages（append reducer）
  intent: Annotation<'consult' | 'apply' | 'ambiguous'>({
    reducer: (_, x) => x,                          // 替换语义
    default: () => 'ambiguous' as const,
  }),
  documentText: Annotation<string>({               // 用户上传的文档文本
    reducer: (_, x) => x,
    default: () => '',
  }),
  confidence: Annotation<'high' | 'medium' | 'low'>({
    reducer: (_, x) => x,
    default: () => 'low' as const,
  }),
})

// 类型快捷方式
type AppStateType = typeof AppState.State

const model = new ChatOpenAI({
  apiKey: process.env.QWEN_API_KEY!,
  configuration: { baseURL: process.env.QWEN_BASE_URL },
  modelName: process.env.QWEN_MODEL,
  temperature: 0,
})

const intentModel = model.withStructuredOutput(
  z.object({
    intent: z.enum(['consult', 'apply', 'ambiguous']),
    confidence: z.enum(['high', 'medium', 'low']),
    reason: z.string(),
  })
)

// 节点：意图分类（现在直接写 state 字段，不用塞 additional_kwargs）
async function classifyNode(state: AppStateType): Promise<Partial<AppStateType>> {
  const lastMsg = state.messages.at(-1)!
  const result = await intentModel.invoke([
    new SystemMessage(`判断用户意图：
      - consult：用户在询问政策、规定、条件等咨询类问题
      - apply：用户要提交材料、申请加分、生成申请表等操作类请求
      - ambiguous：描述不清，无法判断
    回答要简短，reason 不超过15字。`),
    new HumanMessage(String(lastMsg.content)),
  ])
  console.log(`  → 意图: ${result.intent}（${result.confidence}置信度）原因: ${result.reason}`)
  return { intent: result.intent, confidence: result.confidence }  // 直接更新状态字段
}

// 节点：咨询回复（读取 state.intent 和 state.documentText）
async function consultNode(state: AppStateType): Promise<Partial<AppStateType>> {
  const userMsg = state.messages.find(m => m._getType() === 'human')!
  const context = state.documentText ? `\n用户提供的材料：${state.documentText.slice(0, 500)}` : ''
  const reply = await model.invoke([
    new SystemMessage(`你是推免政策咨询助手。${context}`),
    new HumanMessage(String(userMsg.content)),
  ])
  return { messages: [reply] }
}

// 节点：申请引导
async function applyNode(state: AppStateType): Promise<Partial<AppStateType>> {
  const userMsg = state.messages.find(m => m._getType() === 'human')!
  const reply = await model.invoke([
    new SystemMessage('你是申请表单助手。根据用户描述，列出需要准备的材料清单（用 markdown 列表）。'),
    new HumanMessage(String(userMsg.content)),
  ])
  return { messages: [reply] }
}

// 节点：请求澄清
async function clarifyNode(state: AppStateType): Promise<Partial<AppStateType>> {
  return {
    messages: [new AIMessage(
      '您的描述比较模糊，请告诉我：\n1. 您是想**咨询某项政策**的具体规定？\n2. 还是要**申请加分**，需要我帮您准备材料？'
    )]
  }
}

// 路由：直接读 state.intent 字段（不用从消息里挖）
function routeIntent(state: AppStateType) {
  return state.intent
}

const graph = new StateGraph(AppState)
  .addNode('classify', classifyNode)
  .addNode('consult', consultNode)
  .addNode('apply', applyNode)
  .addNode('clarify', clarifyNode)
  .addEdge(START, 'classify')
  .addConditionalEdges('classify', routeIntent, {
    consult: 'consult',
    apply: 'apply',
    ambiguous: 'clarify',
  })
  .addEdge('consult', END)
  .addEdge('apply', END)
  .addEdge('clarify', END)

const app = graph.compile()

// 测试：模拟用户上传了一段文档文本
const result = await app.invoke({
  messages: [new HumanMessage('我有一个全国竞赛的获奖证书')],
  documentText: '第十八届挑战杯全国大学生课外学术科技作品竞赛，二等奖，2024年6月',
})

console.log('\n最终 State:')
console.log('  intent:', result.intent)
console.log('  confidence:', result.confidence)
const lastAI = result.messages.filter((m: { _getType: () => string }) => m._getType() === 'ai').at(-1)
console.log('  回复:', String(lastAI?.content ?? '').slice(0, 120))
```

**验收**：`result.intent`、`result.confidence` 都有值，不再需要从消息里挖 `additional_kwargs`。

**核心理解**：`Annotation.Root` 让你把"对话消息"和"业务状态"放在一个 State 里。节点的返回值是 `Partial<State>`，只写你想更新的字段即可。

### Day 18 — interrupt() + Command（暂停与恢复）

**今天学什么**：`interrupt()` 是 LangGraph 中最关键的能力之一——它可以在任意节点**暂停执行，等待外部输入**（如人工审核、用户补充信息），然后用 `Command({ resume: 答案 })` 恢复继续。

这正是你的"信息不足兜底"场景：Agent 发现信息不够 → 暂停 → 用户补充后 → 继续处理。

**做什么**：实现一个需要用户确认的申请流程，信息不完整时暂停等待补充。

```typescript
// day-18.ts
import 'dotenv/config'
import { StateGraph, Annotation, MessagesAnnotation, START, END, interrupt } from '@langchain/langgraph'
import { Command, MemorySaver } from '@langchain/langgraph'
import { ChatOpenAI } from '@langchain/openai'
import { HumanMessage, AIMessage } from '@langchain/core/messages'
import { z } from 'zod'

const AppState = Annotation.Root({
  ...MessagesAnnotation.spec,
  missingInfo: Annotation<string[]>({
    reducer: (_, x) => x,
    default: () => [],
  }),
  supplemented: Annotation<string>({
    reducer: (_, x) => x,
    default: () => '',
  }),
})

type S = typeof AppState.State

const model = new ChatOpenAI({
  apiKey: process.env.QWEN_API_KEY!,
  configuration: { baseURL: process.env.QWEN_BASE_URL },
  modelName: process.env.QWEN_MODEL,
  temperature: 0,
})

// 节点1：验证信息完整性
async function validateNode(state: S): Promise<Partial<S>> {
  const userMsg = state.messages.find(m => m._getType() === 'human')!
  const result = await model.withStructuredOutput(
    z.object({
      isComplete: z.boolean(),
      missing: z.array(z.string()),
    })
  ).invoke([
    new HumanMessage(`判断以下材料描述是否完整（需包含：赛事名称、奖项等级、时间、申请人角色）：
    "${String(userMsg.content)}"`),
  ])
  console.log('  → 完整性检查:', result)
  return { missingInfo: result.missing }
}

// 节点2：信息不足时 interrupt 暂停等用户补充
// ⚠️ 这个节点执行到 interrupt() 就暂停，等 resume 后才继续
async function askForMoreNode(state: S): Promise<Partial<S>> {
  const question = `材料信息不完整，缺少：${state.missingInfo.join('、')}。请补充这些信息：`
  console.log('  → 暂停，向用户提问:', question)

  // interrupt() 暂停执行，返回值是 resume 时传入的数据
  const userAnswer = interrupt(question)

  console.log('  → 用户补充了:', userAnswer)
  return {
    supplemented: String(userAnswer),
    messages: [new AIMessage(`好的，已记录您补充的信息：${userAnswer}`)],
  }
}

// 节点3：生成申请内容
async function generateNode(state: S): Promise<Partial<S>> {
  const userMsg = state.messages.find(m => m._getType() === 'human')!
  const extra = state.supplemented ? `\n补充信息：${state.supplemented}` : ''
  const reply = await model.invoke([
    new HumanMessage(`根据以下材料信息，生成申请表单草稿：
    原始描述：${String(userMsg.content)}${extra}
    
    请列出：1.申请加分类别 2.预计分值 3.需要提交的材料`),
  ])
  return { messages: [reply] }
}

// 路由：信息完整 → 直接生成；不完整 → 先询问
function checkComplete(state: S) {
  return state.missingInfo.length === 0 ? 'generate' : 'ask'
}

const graph = new StateGraph(AppState)
  .addNode('validate', validateNode)
  .addNode('ask', askForMoreNode)
  .addNode('generate', generateNode)
  .addEdge(START, 'validate')
  .addConditionalEdges('validate', checkComplete, { generate: 'generate', ask: 'ask' })
  .addEdge('ask', 'generate')  // 补充后继续生成
  .addEdge('generate', END)

// ✅ 必须有 checkpointer，interrupt 才能保存断点
const checkpointer = new MemorySaver()
const app = graph.compile({ checkpointer })

const threadConfig = { configurable: { thread_id: 'apply-demo-1' } }

// 第一次调用：信息不完整，会在 ask 节点暂停
console.log('=== 第一次调用（信息不完整）===')
const result1 = await app.invoke(
  { messages: [new HumanMessage('我拿了挑战杯的奖，想申请加分')] },
  threadConfig
)
// result1 会在 interrupt 处暂停，最后一条 AI 消息是提问
const lastMsg1 = result1.messages.at(-1)
console.log('暂停后收到问题:', String(lastMsg1?.content ?? ''))

// ⏸️ 模拟用户去看了信息，过了几秒回来补充
await new Promise(r => setTimeout(r, 500))

// 第二次调用：用 Command({ resume: 答案 }) 恢复，传入用户的补充
console.log('\n=== 第二次调用（用户补充信息，恢复执行）===')
const result2 = await app.invoke(
  new Command({ resume: '全国二等奖，2024年6月，我是队长' }),
  threadConfig
)
const lastMsg2 = result2.messages.filter((m: { _getType: () => string }) => m._getType() === 'ai').at(-1)
console.log('\n最终申请草稿:')
console.log(String(lastMsg2?.content ?? ''))
```

**验收**：
- 第一次调用后程序输出"暂停后收到问题"，说明 interrupt 生效
- 第二次用 `Command({ resume: ... })` 恢复，程序从断点继续，最终输出申请草稿
- 再次运行用一个新的 `thread_id`，历史完全隔离

**核心理解**：
- `interrupt(prompt)` 暂停当前节点，返回值是 resume 时传入的数据
- `new Command({ resume: data })` 作为下次 `invoke` 的输入来恢复
- **必须有 checkpointer**，否则无法保存断点状态

### Day 19 — Send API（并行节点执行）

**今天学什么**：当有多个互不依赖的任务时，可以用 `Send` 把它们同时发给多个节点并行执行，最后合并结果。适用于"同时检查材料的多个属性"或"同时调用多个数据源"。

**做什么**：对一份材料同时做"政策匹配检查"和"格式完整性检查"，两个检查并行执行，最后汇总。

```typescript
// day-19.ts
import 'dotenv/config'
import { StateGraph, Annotation, MessagesAnnotation, START, END, Send } from '@langchain/langgraph'
import { ChatOpenAI } from '@langchain/openai'
import { HumanMessage } from '@langchain/core/messages'
import { z } from 'zod'

// 主状态
const MainState = Annotation.Root({
  ...MessagesAnnotation.spec,
  documentText: Annotation<string>({ reducer: (_, x) => x, default: () => '' }),
  checkResults: Annotation<string[]>({
    reducer: (existing, newVal) => [...existing, ...newVal],  // ✅ 追加，而不是替换！
    default: () => [],
  }),
})

// 子任务状态（每个并行任务的输入）
const CheckTaskState = Annotation.Root({
  documentText: Annotation<string>({ reducer: (_, x) => x, default: () => '' }),
  checkType: Annotation<string>({ reducer: (_, x) => x, default: () => '' }),
})

type MainS = typeof MainState.State
type CheckS = typeof CheckTaskState.State

const model = new ChatOpenAI({
  apiKey: process.env.QWEN_API_KEY!,
  configuration: { baseURL: process.env.QWEN_BASE_URL },
  modelName: process.env.QWEN_MODEL,
  temperature: 0,
})

// 分发节点：用 Send 把任务派给多个并行节点
function dispatchChecks(state: MainS) {
  const checks = ['policy_match', 'format_check', 'time_validity']
  // 每个 Send 都是一个独立的节点调用，并行执行
  return checks.map(checkType => new Send('runCheck', {
    documentText: state.documentText,
    checkType,
  }))
}

// 并行执行的检查节点（每个实例独立运行）
async function runCheckNode(state: CheckS): Promise<Partial<MainS>> {
  const prompts: Record<string, string> = {
    policy_match: `检查这份材料是否符合推免加分政策（50字以内）：${state.documentText}`,
    format_check: `检查这份材料的格式是否完整（包含赛事名称/等级/时间）（50字以内）：${state.documentText}`,
    time_validity: `检查材料的时间是否在有效申请期内（一般要求本科期间）（50字以内）：${state.documentText}`,
  }
  const prompt = prompts[state.checkType] ?? '未知检查类型'
  const result = await model.invoke([new HumanMessage(prompt)])
  const checkResult = `[${state.checkType}] ${String(result.content)}`
  console.log('  完成检查:', checkResult.slice(0, 60))
  return { checkResults: [checkResult] }  // 追加到主 state 的 checkResults 数组
}

// 汇总节点：所有并行任务完成后执行
async function summarizeNode(state: MainS): Promise<Partial<MainS>> {
  const allResults = state.checkResults.join('\n')
  const summary = await model.invoke([
    new HumanMessage(`综合以下检查结果，给出最终评估意见（100字以内）：\n${allResults}`),
  ])
  return { messages: [summary] }
}

const graph = new StateGraph(MainState)
  .addNode('dispatch', dispatchChecks)    // 分发节点（注意：直接是函数，不是 async 节点）
  .addNode('runCheck', runCheckNode)      // 并行执行节点
  .addNode('summarize', summarizeNode)    // 汇总节点
  .addEdge(START, 'dispatch')
  // Send API 的路由写法：第三个参数是可能的目标节点列表
  .addConditionalEdges('dispatch', s => s, ['runCheck'])
  .addEdge('runCheck', 'summarize')
  .addEdge('summarize', END)

const app = graph.compile()

console.log('开始并行检查...')
const result = await app.invoke({
  messages: [new HumanMessage('请检查我的材料')],
  documentText: '第十八届挑战杯全国大学生课外学术科技作品竞赛，二等奖，2024年6月，本科二年级',
})

console.log('\n所有检查结果:')
result.checkResults.forEach((r: string) => console.log(' ', r))
console.log('\n综合评估:')
const lastAI = result.messages.filter((m: { _getType: () => string }) => m._getType() === 'ai').at(-1)
console.log(String(lastAI?.content ?? ''))
```

**验收**：三个检查（policy_match、format_check、time_validity）几乎同时开始打印，而不是顺序执行，总耗时比顺序执行短。

**关键理解**：`checkResults` 的 reducer 是 `(existing, newVal) => [...existing, ...newVal]`（追加），不能用替换，否则并行写入时会互相覆盖。

### Day 20 — 子图（Subgraph）

**今天学什么**：当一个节点内部逻辑很复杂时，可以把它拆成一个独立的子图。子图有自己的状态和节点，对主图来说像一个普通节点一样使用。这样主图保持简洁，每条路径的详细逻辑封装在各自的子图里。

**做什么**：把"咨询路径"封装为一个子图，主图负责意图分类和路由，子图负责咨询的内部多步骤逻辑（检索→回答→验证）。

```typescript
// day-20.ts
import 'dotenv/config'
import { StateGraph, MessagesAnnotation, Annotation, START, END } from '@langchain/langgraph'
import { ChatOpenAI } from '@langchain/openai'
import { HumanMessage, AIMessage, SystemMessage } from '@langchain/core/messages'
import { z } from 'zod'

const model = new ChatOpenAI({
  apiKey: process.env.QWEN_API_KEY!,
  configuration: { baseURL: process.env.QWEN_BASE_URL },
  modelName: process.env.QWEN_MODEL,
  temperature: 0,
})

// ========= 子图：咨询路径 =========
// 子图的状态（必须包含 messages，因为要与主图通信）
const ConsultState = Annotation.Root({
  ...MessagesAnnotation.spec,
  retrievedContext: Annotation<string>({ reducer: (_, x) => x, default: () => '' }),
  answerDraft: Annotation<string>({ reducer: (_, x) => x, default: () => '' }),
})

type ConsultS = typeof ConsultState.State

// 子图节点1：模拟知识库检索（实际项目中调用 vectorStore.ts）
async function retrieveNode(state: ConsultS): Promise<Partial<ConsultS>> {
  const userMsg = state.messages.find(m => m._getType() === 'human')!
  console.log('  [子图] 检索知识库...')
  // 模拟检索结果（Day 22 会替换为真实的 vectorStore 调用）
  const mockContext = `政策条文：挑战杯全国一等奖 6分，二等奖 4.5分，三等奖 3分；
  申请条件：本科在读期间获奖，颁奖机构须为教育部认定赛事。`
  return { retrievedContext: mockContext }
}

// 子图节点2：基于检索内容生成回答
async function answerNode(state: ConsultS): Promise<Partial<ConsultS>> {
  const userMsg = state.messages.find(m => m._getType() === 'human')!
  console.log('  [子图] 生成回答...')
  const reply = await model.invoke([
    new SystemMessage(`你是政策咨询助手。根据以下政策条文回答用户问题：\n${state.retrievedContext}`),
    new HumanMessage(String(userMsg.content)),
  ])
  return { answerDraft: String(reply.content), messages: [reply] }
}

// 构建咨询子图
const consultSubgraph = new StateGraph(ConsultState)
  .addNode('retrieve', retrieveNode)
  .addNode('answer', answerNode)
  .addEdge(START, 'retrieve')
  .addEdge('retrieve', 'answer')
  .addEdge('answer', END)
  .compile()  // 子图也需要 compile

// ========= 主图 =========
const MainState = Annotation.Root({
  ...MessagesAnnotation.spec,
  intent: Annotation<'consult' | 'apply' | 'ambiguous'>({
    reducer: (_, x) => x,
    default: () => 'ambiguous' as const,
  }),
})

type MainS = typeof MainState.State

// 主图节点：意图分类
async function classifyNode(state: MainS): Promise<Partial<MainS>> {
  const lastMsg = state.messages.at(-1)!
  const result = await model.withStructuredOutput(
    z.object({ intent: z.enum(['consult', 'apply', 'ambiguous']) })
  ).invoke([
    new SystemMessage('判断用户意图：consult/apply/ambiguous'),
    new HumanMessage(String(lastMsg.content)),
  ])
  console.log('  [主图] 意图:', result.intent)
  return { intent: result.intent }
}

// 主图节点：直接使用子图作为节点（关键写法！）
// 注意：子图的输入/输出会自动转换为主图的 messages 格式
async function consultRouteNode(state: MainS): Promise<Partial<MainS>> {
  // 调用子图，传入当前消息
  const result = await consultSubgraph.invoke({ messages: state.messages })
  // 把子图的最后一条 AI 消息追加到主图
  const subAI = result.messages.filter((m: { _getType: () => string }) => m._getType() === 'ai').at(-1)
  return { messages: subAI ? [subAI] : [] }
}

async function applyRouteNode(state: MainS): Promise<Partial<MainS>> {
  const userMsg = state.messages.find(m => m._getType() === 'human')!
  const reply = await model.invoke([
    new SystemMessage('你是申请助手，列出申请所需材料清单。'),
    new HumanMessage(String(userMsg.content)),
  ])
  return { messages: [reply] }
}

async function clarifyRouteNode(state: MainS): Promise<Partial<MainS>> {
  return {
    messages: [new AIMessage('您是想**咨询政策规定**，还是要**申请加分**？请描述得更具体一些。')]
  }
}

const mainGraph = new StateGraph(MainState)
  .addNode('classify', classifyNode)
  .addNode('consult', consultRouteNode)  // ← 子图作为节点
  .addNode('apply', applyRouteNode)
  .addNode('clarify', clarifyRouteNode)
  .addEdge(START, 'classify')
  .addConditionalEdges('classify', s => s.intent, {
    consult: 'consult',
    apply: 'apply',
    ambiguous: 'clarify',
  })
  .addEdge('consult', END)
  .addEdge('apply', END)
  .addEdge('clarify', END)

const app = mainGraph.compile()

const result = await app.invoke({
  messages: [new HumanMessage('挑战杯二等奖能加多少分？申请条件是什么？')]
})
const lastAI = result.messages.filter((m: { _getType: () => string }) => m._getType() === 'ai').at(-1)
console.log('\n最终回复:', String(lastAI?.content ?? ''))
```

**验收**：咨询问题经过主图分类后，进入子图执行"检索→回答"两步，日志中看到 `[主图]` 和 `[子图]` 的分层打印。

**关键理解**：子图有自己的 `State` 和节点，对主图来说是透明的。主图和子图通过 `messages` 字段通信（因为两个 State 都扩展自 `MessagesAnnotation.spec`）。

```typescript
// src/agents/subgraphs/consultGraph.ts
import { StateGraph, Annotation, MessagesAnnotation, START, END } from '@langchain/langgraph'
import { ChatOpenAI } from '@langchain/openai'
import { HumanMessage, SystemMessage } from '@langchain/core/messages'
import { getEmbedding } from '../../services/embeddings.js'  // ✅ 复用现有服务
import { searchSimilar } from '../../services/vectorStore.js'  // ✅ 复用现有服务
import { getApiKey, getBaseUrl, getChatModel, getSystemRole } from '../../services/aiConfig.js'

// 子图状态（扩展主图的关键字段）
const ConsultSubState = Annotation.Root({

  const queryEmbedding = await getEmbedding(query)
  const chunks = searchSimilar(queryEmbedding, 5)

  const context = chunks.length > 0

    : '（知识库中暂无相关政策条文）'

  return { retrievedContext: context }
}

// 子图节点2：基于检索结果生成回答
async function ragAnswerNode(state: S): Promise<Partial<S>> {
  const model = new ChatOpenAI({
    apiKey: getApiKey(),
    configuration: { baseURL: getBaseUrl() },
    modelName: getChatModel(),
    temperature: 0.3,
  })

  const systemBase = getSystemRole()
  const userMsg = state.messages.at(-1)!

  // 如果有文档，也一起作为上下文
  const docContext = state.documentText
    ? `\n\n用户提供的材料：${state.documentText.slice(0, 800)}`
    : ''

  const reply = await model.invoke([
    new SystemMessage(
      `${systemBase}\n\n【知识库检索结果】\n${state.retrievedContext}${docContext}\n\n基于以上内容回答用户问题，如知识库无相关内容请说明。`
```

```typescript
// src/agents/nodes/clarifyNode.ts（兜底节点）
import { interrupt } from '@langchain/langgraph'
import { AIMessage } from '@langchain/core/messages'
import type { AgentStateType } from '../states.js'

/**
 * 兜底节点：信息不足时暂停，等用户补充
 * interrupt() 会暂停图的执行，返回值是 resume 时传入的数据
 */
export async function clarifyNode(state: AgentStateType): Promise<Partial<AgentStateType>> {
  // 根据 missingItems 或 intentConfidence 构建问题
  let question: string

  if (state.missingItems.length > 0) {
    question = `我需要更多信息才能帮您：\n缺少：${state.missingItems.join('、')}\n请补充以上信息。`
  } else if (state.intentConfidence === 'low') {
    question = `您的描述比较简短，请问您是想：\n1. **咨询政策**（如"挑战杯能加多少分"）\n2. **申请加分**（如"帮我分析这份证书"）\n\n请描述得更具体一些。`
  } else {
    question = '请提供更多详情，以便我更好地帮助您。'
  }

  // ⏸️ 暂停，等待用户补充
  const userSupplement = interrupt(question)

  // 用户补充后，把补充内容追加到消息，重新路由
  return {
    userInput: String(userSupplement),
    messages: [new AIMessage(`好的，您补充了：${String(userSupplement)}\n我来重新分析...`)],
  }
}
```

```typescript
// src/agents/subgraphs/applyGraph.ts（申请生成子图）
import { StateGraph, Annotation, MessagesAnnotation, START, END } from '@langchain/langgraph'
import { ChatOpenAI } from '@langchain/openai'
import { HumanMessage, SystemMessage } from '@langchain/core/messages'
import { z } from 'zod'
import { getEmbedding } from '../../services/embeddings.js'
import { searchSimilar } from '../../services/vectorStore.js'
import { getApiKey, getBaseUrl, getChatModel } from '../../services/aiConfig.js'

const ApplySubState = Annotation.Root({
  ...MessagesAnnotation.spec,
  documentText: Annotation<string>({ reducer: (_, x) => x, default: () => '' }),
  missingItems: Annotation<string[]>({ reducer: (_, x) => x, default: () => [] }),
  policyContext: Annotation<string>({ reducer: (_, x) => x, default: () => '' }),
})

type S = typeof ApplySubState.State

// 节点1：检索相关政策（基于文档内容）
async function fetchPolicyNode(state: S): Promise<Partial<S>> {
  const query = state.documentText.slice(0, 200) || '加分政策'
  const embedding = await getEmbedding(query)
  const chunks = searchSimilar(embedding, 5)
  const policyContext = chunks.map(c => c.content).join('\n\n')
  return { policyContext }
}

// 节点2：验证材料完整性
async function validateMaterialNode(state: S): Promise<Partial<S>> {
  const model = new ChatOpenAI({
    apiKey: getApiKey(), configuration: { baseURL: getBaseUrl() },
    modelName: getChatModel(), temperature: 0,
  }).withStructuredOutput(z.object({
    isComplete: z.boolean(),
    missingItems: z.array(z.string()),
  }))

  const result = await model.invoke([
    new HumanMessage(`检查材料是否包含：赛事名称、奖项等级、颁奖时间、申请人身份\n材料：${state.documentText}`),
  ])
  return { missingItems: result.missingItems }
}

// 节点3：生成申请建议
async function generateSuggestionNode(state: S): Promise<Partial<S>> {
  const model = new ChatOpenAI({
    apiKey: getApiKey(), configuration: { baseURL: getBaseUrl() },
    modelName: getChatModel(), temperature: 0.3,
  })
  const reply = await model.invoke([
    new SystemMessage(`你是推免加分申请助手。根据材料和政策，生成结构化的申请建议。
    相关政策：${state.policyContext.slice(0, 1000)}`),
    new HumanMessage(`请基于以下材料生成申请建议：\n${state.documentText}`),
  ])
  return { messages: [reply] }
}

// 路由：材料完整 → 直接生成；不完整 → 先标记缺失项（由主图的 clarify 节点处理）
function checkMaterial(state: S) {
  return state.missingItems.length === 0 ? 'generate' : 'incomplete'
}

export const applySubgraph = new StateGraph(ApplySubState)
  .addNode('fetchPolicy', fetchPolicyNode)
  .addNode('validate', validateMaterialNode)
  .addNode('generate', generateSuggestionNode)
  .addEdge(START, 'fetchPolicy')
  .addEdge('fetchPolicy', 'validate')
  .addConditionalEdges('validate', checkMaterial, {
    generate: 'generate',
    incomplete: END,  // 不完整时直接结束，由主图的 clarify 节点接管
  })
  .addEdge('generate', END)
  .compile()
```

**验收**：
- 传入完整材料（有赛事名/等级/时间）→ 走到 generate 节点，输出申请建议
- 传入不完整材料 → missingItems 不为空，返回缺失项列表

### Day 25 — 主图整合 + checkpointer + 流式输出（项目实战④）

**新建文件**：`src/agents/mainAgent.ts`

**今天做什么**：把前三天的节点和子图整合成完整的主图，接入 MemorySaver，支持多轮对话和 interrupt 恢复。

```typescript
// src/agents/mainAgent.ts
import { StateGraph, START, END, MemorySaver } from '@langchain/langgraph'
import { Command } from '@langchain/langgraph'
import { AgentState, type AgentStateType } from './states.js'
import { parseInputNode } from './nodes/parseInputNode.js'
import { classifyIntentNode } from './nodes/classifyIntentNode.js'
import { clarifyNode } from './nodes/clarifyNode.js'
import { consultSubgraph } from './subgraphs/consultGraph.js'
import { applySubgraph } from './subgraphs/applyGraph.js'
import { AIMessage } from '@langchain/core/messages'

// 适配节点：把子图返回的 messages 合并到主图
async function runConsultNode(state: AgentStateType): Promise<Partial<AgentStateType>> {
  const result = await consultSubgraph.invoke({
    messages: state.messages,
    userInput: state.userInput,
    documentText: state.documentText,
  })
  const subAI = result.messages.filter((m: { _getType: () => string }) => m._getType() === 'ai').at(-1)
  return { messages: subAI ? [subAI] : [] }
}

async function runApplyNode(state: AgentStateType): Promise<Partial<AgentStateType>> {
  const result = await applySubgraph.invoke({
    messages: state.messages,
    documentText: state.documentText,
  })
  const subAI = result.messages.filter((m: { _getType: () => string }) => m._getType() === 'ai').at(-1)
  const missing = result.missingItems as string[] | undefined
  return {
    messages: subAI ? [subAI] : [],
    missingItems: missing ?? [],
  }
}

// 路由函数：意图 + 申请后的材料完整性检查
function routeAfterClassify(state: AgentStateType) {
  return state.intent  // → 'consult' | 'apply' | 'ambiguous'
}

function routeAfterApply(state: AgentStateType) {
  // 申请子图运行后，若有缺失项，走 clarify；否则结束
  return state.missingItems.length > 0 ? 'clarify' : END
}

// 构建主图
const checkpointer = new MemorySaver()

const mainGraph = new StateGraph(AgentState)
  .addNode('parseInput', parseInputNode)
  .addNode('classify', classifyIntentNode)
  .addNode('consult', runConsultNode)
  .addNode('apply', runApplyNode)
  .addNode('clarify', clarifyNode)           // interrupt 节点
  .addEdge(START, 'parseInput')
  .addEdge('parseInput', 'classify')
  .addConditionalEdges('classify', routeAfterClassify, {
    consult: 'consult',
    apply: 'apply',
    ambiguous: 'clarify',
  })
  .addEdge('consult', END)
  .addConditionalEdges('apply', routeAfterApply, {
    clarify: 'clarify',
    [END]: END,
  })
  // clarify 节点 interrupt 后 resume → 重新分类
  .addEdge('clarify', 'classify')

export const agent = mainGraph.compile({ checkpointer })

// ========= 供路由层调用的两个接口 =========

export interface AgentInput {
  userInput: string
  documentText?: string
  sessionId: string             // 对应 thread_id
}

/** 普通调用（非流式） */
export async function invokeAgent(input: AgentInput) {
  const threadConfig = { configurable: { thread_id: input.sessionId } }
  const result = await agent.invoke({
    userInput: input.userInput,
    documentText: input.documentText ?? '',
  }, threadConfig)

  const lastAI = result.messages
    .filter((m: { _getType: () => string }) => m._getType() === 'ai')
    .at(-1)
  return {
    reply: String(lastAI?.content ?? ''),
    intent: result.intent,
    interrupted: false,
  }
}

/** Resume 调用（用户补充信息后恢复） */
export async function resumeAgent(sessionId: string, supplement: string) {
  const threadConfig = { configurable: { thread_id: sessionId } }
  const result = await agent.invoke(
    new Command({ resume: supplement }),
    threadConfig
  )
  const lastAI = result.messages
    .filter((m: { _getType: () => string }) => m._getType() === 'ai')
    .at(-1)
  return {
    reply: String(lastAI?.content ?? ''),
    intent: result.intent,
  }
}

/** 流式调用（SSE） */
export async function* streamAgent(input: AgentInput): AsyncGenerator<string> {
  const threadConfig = { configurable: { thread_id: input.sessionId } }
  const stream = await agent.stream({
    userInput: input.userInput,
    documentText: input.documentText ?? '',
  }, { ...threadConfig, streamMode: 'values' })

  let lastContent = ''
  for await (const state of stream) {
    const msgs = state.messages ?? []
    const lastAI = msgs.filter((m: { _getType: () => string }) => m._getType() === 'ai').at(-1)
    const content = String(lastAI?.content ?? '')
    if (content && content !== lastContent) {
      // 只 yield 新增的部分
      const delta = content.slice(lastContent.length)
      if (delta) yield delta
      lastContent = content
    }
  }
}
```

**验收**：
- 调用 `invokeAgent`，传入咨询问题，能收到 RAG 回答
- 传入模糊输入，能感知到 interrupt（可通过捕获异常或检查 state 来判断）
- 调用 `resumeAgent`，传入补充信息，图从断点继续执行

### Day 26 — 接入 Express 路由（项目实战⑤，完成！）

**修改文件**：`src/routes/chat.ts`（新增 LangGraph Agent 路由）

**今天做什么**：把 `mainAgent.ts` 接入 Express 路由，替代 `ragChain.ts` 的旧路由，支持文件上传和流式输出。

```typescript
// src/routes/chat.ts（新增路由，保留旧路由作为备份）
import { Router } from 'express'
import multer from 'multer'
import { parseBuffer } from '../services/docParser.js'
import { invokeAgent, resumeAgent, streamAgent } from '../agents/mainAgent.js'

const router = Router()
const upload = multer({ storage: multer.memoryStorage(), limits: { fileSize: 10 * 1024 * 1024 } })

/**
 * POST /chat/agent
 * 新的统一入口：支持文本 + 文件，自动意图分类
 * Body（multipart/form-data）：
 *   - message: string（用户文字）
 *   - sessionId: string
 *   - file?: File（可选，PDF/DOCX/图片）
 */
router.post('/agent', upload.single('file'), async (req, res) => {
  try {
    const { message = '', sessionId = 'default' } = req.body as { message?: string; sessionId?: string }

    // 解析上传的文件（如果有）
    let documentText = ''
    if (req.file) {
      documentText = await parseBuffer(req.file.buffer, req.file.originalname)
    }

    const result = await invokeAgent({ userInput: message, documentText, sessionId })
    res.json({ reply: result.reply, intent: result.intent })
  } catch (err) {
    res.status(500).json({ error: String(err) })
  }
})

/**
 * POST /chat/agent/stream
 * 流式版本（SSE）
 */
router.post('/agent/stream', upload.single('file'), async (req, res) => {
  const { message = '', sessionId = 'default' } = req.body as { message?: string; sessionId?: string }

  let documentText = ''
  if (req.file) {
    documentText = await parseBuffer(req.file.buffer, req.file.originalname)
  }

  res.setHeader('Content-Type', 'text/event-stream')
  res.setHeader('Cache-Control', 'no-cache')
  res.setHeader('Connection', 'keep-alive')

  try {
    for await (const token of streamAgent({ userInput: message, documentText, sessionId })) {
      res.write(`data: ${JSON.stringify({ token })}\n\n`)
    }
    res.write('data: [DONE]\n\n')
  } catch (err) {
    res.write(`data: ${JSON.stringify({ error: String(err) })}\n\n`)
  } finally {
    res.end()
  }
})

/**
 * POST /chat/agent/resume
 * interrupt 后用户补充信息，恢复执行
 * Body: { sessionId: string, supplement: string }
 */
router.post('/agent/resume', async (req, res) => {
  const { sessionId, supplement } = req.body as { sessionId: string; supplement: string }
  const result = await resumeAgent(sessionId, supplement)
  res.json({ reply: result.reply, intent: result.intent })
})

export default router
```

## 关键设计决策备忘

### 关于 RAG 数据库

你现有的 `vectorStore.ts`（SQLite + 手写余弦相似度）**不需要换**。它就是真实数据库，Day 23 直接调用 `getEmbedding()` + `searchSimilar()` 接入 LangGraph 节点即可。

换 pgvector/Chroma 的时机：知识库 chunk 数量超过 **5 万条**或需要多机部署时再考虑。

### 关于历史记录

原 `ragChain.ts` 用 SQLite 手写 `getHistory()` / `saveMessage()`。接入 LangGraph 后，**用 `MemorySaver` 替代**，不再需要手写历史管理。如果需要持久化到重启（MemorySaver 是内存），可以实现 `BaseCheckpointSaver` 接口，把 checkpoint 存到 SQLite。

### 关于 interrupt 的前端处理

当后端触发 interrupt 时，`invokeAgent` 返回一个特殊的 `interrupted: true` 标志（需要在路由层检测图的状态）。前端收到这个标志后，展示一个补充信息的输入框，用户提交后调用 `/chat/agent/resume`。

### 关于 LCEL 和 LangGraph 的共存

不需要删除旧代码。`/analyze/certificate` 路由可以继续用旧的 LCEL chain 或按 `02阶段2` 文档升级为 Agent。新的 `/chat/agent` 是独立路由。两套系统可以并存，逐步迁移。

## 常见问题备查（进阶版）

**Q：interrupt() 和普通的"让模型反问"有什么区别？**

普通反问：模型在消息里说"请补充信息"，用户下次发消息时图从头开始，不记得上次的状态。
`interrupt()`：图在当前节点**暂停**，所有状态由 checkpointer 保存。用户补充后，图从**断点处**继续，不需要重新分类意图。

**Q：为什么 MessagesAnnotation 的 reducer 是 append 而不是 replace？**

因为对话历史需要不断累积。如果 reducer 是替换，每次节点返回消息都会清空之前的对话记录。append 确保每次只追加新消息，整个推理链都保留。

**Q：子图和主图的 State 必须相同吗？**

不需要。子图有自己的 State，只要**包含 `messages` 字段**，主图就能把当前消息传给子图，并从子图结果中提取 AI 消息。字段名不同的部分，需要在适配节点里手动转换。

**Q：`Send` 的并行和 `Promise.all` 有什么区别？**

都是并行执行。`Send` 的优势是：每个并行任务的状态变化都经过 reducer 合并，并且可以用 checkpointer 持久化中间状态。`Promise.all` 是纯代码层面的并发，没有图的状态管理。

**Q：Day 22–26 的代码需要 `npm install` 新包吗？**

不需要。你的项目已经有 `@langchain/langgraph`（Day 15 起使用）。唯一可能需要的是 `multer`（文件上传中间件），检查 `package.json` 里是否已有：
```bash
npm list multer  # 如果没有：npm install multer @types/multer
```

### 一、 论文写作要求



#### 1. 写作目标

- **学术目标：**
  - 深入研究并阐述基于 Vue 3 和 Spring Boot 的前后端分离架构在教育类管理系统中的设计与实现方法。
  - 分析并探讨 RBAC（基于角色的访问控制）模型在复杂权限系统中的应用，以及如何保障系统数据安全。
  - 探索在管理系统中集成 AI Agent 技术，提升用户交互体验和智能化水平的可行性与实现路径。
  - 总结 MinIO 对象存储和 Redis 缓存技术在提升系统性能和数据管理效率方面的实践经验。
- **应用目标：**
  - 设计并实现一个功能完整、界面友好的 “保研助手” 系统，解决传统保研信息管理流程中存在的信息分散、操作繁琐、效率低下等问题。
  - 实现管理员对保研模板的动态创建与管理，以及学生对模板的高效使用与申请，规范保研申请流程。
  - 提供学生信息列表的导出、排序等实用功能，满足不同用户的多样化需求。
  - 最终交付一个稳定、可靠、可维护的原型系统，为高校保研工作提供信息化支持。

#### 2. 写作思路

写作思路是你完成论文的逻辑框架，可以按照软件工程的标准流程来组织。

1. **绪论 (Introduction):**

   - **研究背景与意义：** 阐述当前高校保研工作的现状、痛点（如信息不透明、材料多、统计难），引出开发 “保研助手” 系统的必要性和价值。
   - **国内外研究现状：** 简要介绍类似教育管理系统的发展情况，以及 Vue 3、Spring Boot 等技术栈的应用现状。
   - **论文主要工作与结构安排：** 概述你在论文中完成的核心工作，并简要介绍论文各章节的内容。

2. **相关技术与理论基础 (Related Technologies and Theoretical Basis):**

   - **前端技术栈：** 详细介绍 Vue 3 的核心特性（如 Composition API、响应式原理）、Vue Router、Pinia（状态管理）、Element Plus（UI 框架）等。
   - **后端技术栈：** 详细介绍 Spring Boot 的核心优势、Spring MVC、MyBatis-Plus（数据持久化）、Spring Security（或 Sa-Token 等用于实现 RBAC）。
   - **中间件：** 介绍 MinIO 作为对象存储服务的原理和优势（用于存储学生上传的附件、导出的 Excel 等），以及 Redis 作为缓存和分布式锁的应用场景（如缓存热门数据、防止重复提交）。
   - **AI Agent 集成：** 介绍你所使用的 AI Agent 技术（如调用大模型 API），以及它在系统中的具体应用场景（如智能问答、材料润色建议等）。
   - **核心理论：** 详细阐述 RBAC 权限模型的原理，并说明如何在你的系统中进行设计和实现。

3. **系统分析 (System Analysis):**

   - 需求分析：

      

     采用用例图、用户故事等方式，详细描述管理员和学生两类角色的功能需求。例如：

     - 管理员：登录、创建 / 修改 / 删除保研模板、管理学生账号、导出学生信息列表。
     - 学生：登录、查看 / 选择保研模板、填写申请信息、提交申请、查询申请状态、使用 AI 助手。

   - **非功能性需求分析：** 分析系统的性能、安全性、易用性、可扩展性等方面的要求。

4. **系统设计 (System Design):**

   - **总体架构设计：** 绘制系统的总体架构图，清晰展示前端、后端、数据库、中间件之间的交互关系。
   - **数据库设计：** 设计系统的 E-R 图，并详细列出核心数据表的结构（字段名、类型、约束、说明）。核心表可能包括：`users`（用户表）、`roles`（角色表）、`permissions`（权限表）、`templates`（模板表）、`template_items`（模板项表）、`applications`（申请表）、`application_details`（申请详情表）等。
   - 核心模块设计：
     - **模板管理模块：** 描述模板的动态创建、数据结构设计、如何关联到申请表单。
     - **申请流程模块：** 描述学生如何根据模板生成并提交申请，以及数据如何在系统中流转。
     - **RBAC 权限模块：** 描述如何设计用户、角色、权限之间的关联关系，并实现动态的权限控制。
     - **AI 助手模块：** 描述 AI Agent 的集成方案、API 调用流程、以及与前端的交互设计。

5. **系统实现 (System Implementation):**

   - 这是论文的核心技术部分。按照模块划分，详细介绍关键功能的实现过程。
   - **前端实现：** 展示关键页面的 Vue 组件代码片段，特别是模板动态渲染、表单提交、AI 交互等逻辑。
   - **后端实现：** 展示核心 Controller、Service、Mapper 层的代码片段，特别是模板 CRUD、申请处理、权限校验、文件上传下载、Redis 缓存使用等关键逻辑。
   - **代码说明：** 对关键代码进行详细注释和解释，说明你为什么这么设计，解决了什么问题。

6. **系统测试 (System Testing):**

   - **测试环境：** 描述测试所使用的软硬件环境。
   - **功能测试：** 设计测试用例，对系统的主要功能点（如模板创建、学生申请、权限控制）进行测试，并记录测试结果。
   - **性能测试（可选）：** 对系统的关键接口进行压力测试，分析响应时间、吞吐量等指标。
   - **安全性测试（可选）：** 测试系统是否能抵御常见的安全攻击，如 SQL 注入、XSS 等。
   - **测试结果分析：** 总结测试过程中发现的问题及修复情况，评估系统是否达到预期目标。

7. **总结与展望 (Conclusion and Future Work):**

   - **工作总结：** 总结你在毕业设计过程中完成的主要工作、遇到的挑战以及解决方案。
   - **系统评价：** 客观评价你所开发系统的优点和不足之处。
   - **未来展望：** 对系统的未来发展方向提出展望，例如：支持更多的 AI 功能、增加移动端适配、与学校其他系统（如教务系统）对接等。

#### 3. 写作方法

- **文献研究法：** 通过查阅国内外相关文献，了解教育管理系统和相关技术的发展动态，为论文的研究提供理论基础。
- **需求分析法：** 通过与潜在用户（老师、同学）沟通，或分析现有流程，明确系统的功能需求和非功能需求。
- **系统设计法：** 运用软件工程的理论和方法，对系统进行总体设计、详细设计，包括架构设计、数据库设计和模块设计。
- **编码实现法：** 根据设计方案，采用 Vue 3、Spring Boot 等技术栈进行系统的编码实现。
- **测试验证法：** 通过设计测试用例，对系统进行全面的功能测试和性能测试，验证系统的正确性和稳定性。

#### 4. 进度安排

以下是一个典型的 16 周毕业设计进度安排，你可以根据实际情况调整：

- **第 1-2 周：** 确定选题，完成开题报告。查阅相关文献，明确研究目标和技术路线。
- **第 3-4 周：** 进行详细的需求分析。与导师和潜在用户沟通，撰写需求规格说明书。
- **第 5-6 周：** 完成系统总体设计和数据库设计。绘制架构图、E-R 图，设计数据表结构。
- **第 7-10 周：** 进行系统编码实现。优先完成核心功能模块（如模板管理、申请流程）。
- **第 11-12 周：** 完成系统的前后端联调，并开始撰写毕业论文的初稿。
- **第 13-14 周：** 进行系统测试，修复发现的 Bug。完善论文内容，补充代码截图和说明。
- **第 15 周：** 完成毕业论文的终稿，准备答辩 PPT。
- **第 16 周：** 进行毕业设计答辩。

### 二、 支持条件

这部分需要导师或学校提供，你可以向导师提出以下需求：

- **硬件支持：** 可提供用于开发和测试的计算机设备。
- **软件支持：** 可提供开发所需的操作系统、开发工具（如 IntelliJ IDEA, VS Code）、数据库软件（MySQL）等。
- **网络支持：** 可提供稳定的校园网络环境，以便于查阅文献、使用在线 API 服务（如 AI Agent）。
- **文献资料支持：** 可提供学校图书馆的数字资源访问权限，方便查阅相关学术论文和技术文档。
- **技术指导支持：** 导师可在关键技术难点上提供指导和建议，帮助解决开发过程中遇到的问题。
- **数据支持：** 可提供（或协助模拟）部分测试数据，如学生信息、课程成绩等，用于系统功能的演示和测试。

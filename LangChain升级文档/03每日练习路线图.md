# LangChain Agent 每日练习路线图

> **前提**：TypeScript + Node.js，模型使用 Qwen3（DashScope，OpenAI 兼容接口）。
> **目标**：21 天，从零写出一个具备工具调用、记忆、流式输出的完整 Agent。
> **原则**：每天只建一个独立文件 `day-XX.ts`，跑通即算完成，不追求完美。

---

## 环境初始化（第一次使用）

在任意空目录执行：

```bash
mkdir langchain-daily && cd langchain-daily
npm init -y
npm install langchain @langchain/openai @langchain/langgraph @langchain/core zod dotenv tsx
```

根目录创建 `.env`：

```env
QWEN_API_KEY=你的DashScope密钥
QWEN_BASE_URL=https://dashscope.aliyuncs.com/compatible-mode/v1
QWEN_MODEL=qwen3-max
```

创建 `tsconfig.json`：

```json
{
  "compilerOptions": {
    "target": "ES2020",
    "module": "ES2020",
    "moduleResolution": "node",
    "strict": true,
    "esModuleInterop": true
  }
}
```

每天运行方式：

```bash
npx tsx day-01.ts
```

## 第一周：核心三件套（模型 · 工具 · Agent）

### Day 01 — 模型最简调用

**今天学什么**：理解 `ChatOpenAI`、`HumanMessage`/`SystemMessage`/`AIMessage` 三种消息类型，以及 `model.invoke()` 的返回值结构。

**做什么**：让模型回答一个问题，打印完整的 `AIMessage` 对象（不只是 content）。

```typescript
// day-01.ts
import 'dotenv/config'
import { ChatOpenAI } from '@langchain/openai'
import { HumanMessage, SystemMessage } from '@langchain/core/messages'

const model = new ChatOpenAI({
  apiKey: process.env.QWEN_API_KEY!,
  configuration: { baseURL: process.env.QWEN_BASE_URL },
  modelName: process.env.QWEN_MODEL,
  temperature: 0.7,
})

const response = await model.invoke([
  new SystemMessage('你是一个简洁的助手，回答不超过30字。'),
  new HumanMessage('TypeScript 相比 JavaScript 最大的优势是什么？'),
])

console.log('内容:', response.content)
console.log('token消耗:', response.response_metadata)
console.log('消息类型:', response._getType()) // 应输出 "ai"
```

**验收**：能看到模型回复 + `tokenUsage` 里的 token 数字。

**踩坑提示**：DashScope 的 `baseURL` 末尾不要加 `/v1`，LangChain 会自动拼接路径。

### Day 02 — 流式输出

**今天学什么**：`model.stream()` 与 `model.invoke()` 的区别，以及如何消费异步生成器。

**做什么**：让模型流式介绍自己，在终端看到逐字打印效果。

```typescript
// day-02.ts
import 'dotenv/config'
import { ChatOpenAI } from '@langchain/openai'
import { HumanMessage } from '@langchain/core/messages'

const model = new ChatOpenAI({
  apiKey: process.env.QWEN_API_KEY!,
  configuration: { baseURL: process.env.QWEN_BASE_URL },
  modelName: process.env.QWEN_MODEL,
})

process.stdout.write('模型回复：')

const stream = await model.stream([
  new HumanMessage('用50字介绍一下你自己。'),
])

for await (const chunk of stream) {
  process.stdout.write(String(chunk.content))  // 逐 token 打印，不换行
}

process.stdout.write('\n')
```

**验收**：终端出现逐字显示效果，不是一次性输出整段。

**思考**：`chunk.content` 有时是空字符串，为什么？（提示：tool_calls chunk）

### Day 03 — 创建第一个 Tool（天气查询）

**今天学什么**：`tool()` 函数的三要素：执行函数、`name`、Zod `schema`。理解 Tool 只是一个带类型描述的普通函数。

**做什么**：创建一个模拟天气查询工具，然后用 `model.bindTools()` 把工具绑定到模型，观察模型返回的 `tool_calls`（此时工具还没有真正执行）。

```typescript
// day-03.ts
import 'dotenv/config'
import { ChatOpenAI } from '@langchain/openai'
import { HumanMessage } from '@langchain/core/messages'
import { tool } from '@langchain/core/tools'
import { z } from 'zod'

// 1. 定义工具
const getWeatherTool = tool(
  (input) => {
    // 模拟数据，实际可接真实天气 API
    const weatherMap: Record<string, string> = {
      '北京': '晴，25°C，微风',
      '上海': '多云，22°C，东南风',
      '厦门': '晴，28°C，海风',
    }
    return weatherMap[input.city] ?? `${input.city}：暂无天气数据`
  },
  {
    name: 'get_weather',
    description: '查询指定城市的当前天气情况',
    schema: z.object({
      city: z.string().describe('城市名称，如：北京、上海'),
    }),
  }
)

// 2. 绑定工具到模型（模型现在"知道"这个工具的存在）
const modelWithTools = new ChatOpenAI({
  apiKey: process.env.QWEN_API_KEY!,
  configuration: { baseURL: process.env.QWEN_BASE_URL },
  modelName: process.env.QWEN_MODEL,
  temperature: 0,
}).bindTools([getWeatherTool])

// 3. 调用模型，观察它是否想调用工具
const response = await modelWithTools.invoke([
  new HumanMessage('厦门今天天气怎么样？'),
])

console.log('模型回复:', response.content)
console.log('工具调用请求:', JSON.stringify(response.tool_calls, null, 2))
// 此时工具还没执行！只是模型表达了"我想调用这个工具"
```

**验收**：`tool_calls` 数组中出现 `{ name: "get_weather", args: { city: "厦门" } }`。

**关键理解**：`bindTools` 只是让模型知道工具存在，不会自动执行。谁来真正调用工具？→ Day 04。

### Day 04 — 手动执行工具（理解 Agent 底层）

**今天学什么**：手动实现一轮"模型→工具→模型"的循环，理解 `ToolMessage` 的作用。这是 Agent 底层运行机制，理解它才能真正理解 Agent。

**做什么**：在 Day 03 基础上，接收模型的 `tool_calls`，手动执行工具，把结果作为 `ToolMessage` 送回给模型，获得最终回复。

```typescript
// day-04.ts
import 'dotenv/config'
import { ChatOpenAI } from '@langchain/openai'
import { HumanMessage, ToolMessage } from '@langchain/core/messages'
import { tool } from '@langchain/core/tools'
import { z } from 'zod'

const getWeatherTool = tool(
  (input) => {
    const data: Record<string, string> = { '厦门': '晴，28°C，海风，湿度65%' }
    return data[input.city] ?? '暂无数据'
  },
  {
    name: 'get_weather',
    description: '查询指定城市的当前天气',
    schema: z.object({ city: z.string() }),
  }
)

const model = new ChatOpenAI({
  apiKey: process.env.QWEN_API_KEY!,
  configuration: { baseURL: process.env.QWEN_BASE_URL },
  modelName: process.env.QWEN_MODEL,
  temperature: 0,
}).bindTools([getWeatherTool])

const messages = [new HumanMessage('厦门今天天气怎么样？')]

// 第一轮：模型决定调用工具
const firstResponse = await model.invoke(messages)
console.log('第一轮 - 工具调用:', firstResponse.tool_calls)

// 手动执行工具
const toolResults = await Promise.all(
  (firstResponse.tool_calls ?? []).map(async (tc) => {
    const result = await getWeatherTool.invoke(tc.args)
    return new ToolMessage({
      content: String(result),
      tool_call_id: tc.id!,  // 必须与 tool_call 的 id 匹配！
    })
  })
)

// 第二轮：把工具结果送回模型，得到最终回复
const finalResponse = await model.invoke([
  ...messages,
  firstResponse,   // AI 消息（包含 tool_calls）
  ...toolResults,  // 工具执行结果
])

console.log('最终回复:', finalResponse.content)
```

**验收**：最终输出类似"厦门今天天气晴朗，气温28°C，海风，湿度65%"。

**关键理解**：`tool_call_id` 是 ToolMessage 与对应 tool_call 的配对凭证，必须一致。

### Day 05 — createAgent（让框架自动循环）

**今天学什么**：`createAgent` 是 Day 04 手动循环的自动化版本。理解 Agent 的输入输出格式：输入是 `messages`，输出也是 `messages`（包含整个推理链）。

**做什么**：用 `createAgent` 重写 Day 04，让框架自动处理工具调用循环。

```typescript
// day-05.ts
import 'dotenv/config'
import { createReactAgent } from '@langchain/langgraph/prebuilt'
import { ChatOpenAI } from '@langchain/openai'
import { HumanMessage } from '@langchain/core/messages'
import { tool } from '@langchain/core/tools'
import { z } from 'zod'

const getWeatherTool = tool(
  (input) => {
    const data: Record<string, string> = {
      '厦门': '晴，28°C',
      '北京': '阴，18°C',
    }
    return data[input.city] ?? '暂无数据'
  },
  {
    name: 'get_weather',
    description: '查询指定城市的天气',
    schema: z.object({ city: z.string() }),
  }
)

const model = new ChatOpenAI({
  apiKey: process.env.QWEN_API_KEY!,
  configuration: { baseURL: process.env.QWEN_BASE_URL },
  modelName: process.env.QWEN_MODEL,
  temperature: 0,
})

// createReactAgent 是 LangGraph 提供的开箱即用 Agent
const agent = createReactAgent({
  llm: model,
  tools: [getWeatherTool],
  stateModifier: '你是天气助手，只回答天气相关问题，回答简洁。',
})

const result = await agent.invoke({
  messages: [new HumanMessage('厦门和北京哪个城市今天更热？')],
})

// result.messages 包含完整的推理链
for (const msg of result.messages) {
  console.log(`[${msg._getType()}]`, String(msg.content).slice(0, 80))
}
```

**验收**：`result.messages` 中依次出现 `human → ai（tool_calls）→ tool → ai（最终回复）`。

**对比 Day 04**：代码从 40 行减到 15 行，且自动处理多工具、多轮调用。

### Day 06 — 多工具 + 工具选择

**今天学什么**：当 Agent 有多个工具时，如何依靠 `description` 引导模型选择正确的工具。`description` 是工具的"身份证"，写好它比写代码更重要。

**做什么**：给 Agent 加上"查汇率"和"查时间"两个额外工具，提问时故意混入多种需求，观察工具选择情况。

```typescript
// day-06.ts
import 'dotenv/config'
import { createReactAgent } from '@langchain/langgraph/prebuilt'
import { ChatOpenAI } from '@langchain/openai'
import { HumanMessage } from '@langchain/core/messages'
import { tool } from '@langchain/core/tools'
import { z } from 'zod'

const getWeatherTool = tool(
  (input) => `${input.city}：晴，28°C`,
  {
    name: 'get_weather',
    description: '查询城市天气。仅用于天气相关问题，不用于汇率或时间查询。',
    schema: z.object({ city: z.string().describe('城市名') }),
  }
)

const getExchangeRateTool = tool(
  (input) => `1 ${input.from} = ${(Math.random() * 2 + 6).toFixed(4)} ${input.to}（模拟数据）`,
  {
    name: 'get_exchange_rate',
    description: '查询两种货币之间的汇率。仅用于货币兑换相关问题。',
    schema: z.object({
      from: z.string().describe('源货币，如 USD'),
      to: z.string().describe('目标货币，如 CNY'),
    }),
  }
)

const getCurrentTimeTool = tool(
  () => `当前时间：${new Date().toLocaleString('zh-CN')}`,
  {
    name: 'get_current_time',
    description: '获取当前的日期和时间。',
    schema: z.object({}),
  }
)

const agent = createReactAgent({
  llm: new ChatOpenAI({
    apiKey: process.env.QWEN_API_KEY!,
    configuration: { baseURL: process.env.QWEN_BASE_URL },
    modelName: process.env.QWEN_MODEL,
    temperature: 0,
  }),
  tools: [getWeatherTool, getExchangeRateTool, getCurrentTimeTool],
})

// 一个问题涉及多个工具
const result = await agent.invoke({
  messages: [new HumanMessage('现在几点了？厦门天气怎么样？1美元等于多少人民币？')],
})

console.log('\n完整推理链:')
for (const msg of result.messages) {
  const type = msg._getType()
  const content = String(msg.content).slice(0, 60)
  const toolName = 'name' in msg ? ` (${(msg as {name:string}).name})` : ''
  console.log(`  [${type}${toolName}] ${content}`)
}
```

**验收**：推理链中出现三个不同的 tool 调用（顺序可能不同）。

**实验**：故意把 `get_exchange_rate` 的 description 改成模糊的"获取数据"，观察模型是否还能正确选择。

### Day 07 — Structured Output（Zod 强制输出格式）

**今天学什么**：`model.withStructuredOutput(ZodSchema)` 的工作原理，以及它与 `tool()` 的区别（structured output 是给最终结果定型，tool 是给中间动作定义）。

**做什么**：让模型分析一段文字，强制输出包含 `sentiment`（正/负/中性）、`score`（0-100）、`keywords` 数组的 JSON 对象。

```typescript
// day-07.ts
import 'dotenv/config'
import { ChatOpenAI } from '@langchain/openai'
import { HumanMessage, SystemMessage } from '@langchain/core/messages'
import { z } from 'zod'

// 定义期望的输出结构
const SentimentSchema = z.object({
  sentiment: z.enum(['positive', 'negative', 'neutral']).describe('情感倾向'),
  score: z.number().min(0).max(100).describe('情感分数，0=极负面，100=极正面'),
  keywords: z.array(z.string()).describe('关键词，不超过5个'),
  summary: z.string().describe('一句话总结，不超过30字'),
})

// 绑定结构化输出
const model = new ChatOpenAI({
  apiKey: process.env.QWEN_API_KEY!,
  configuration: { baseURL: process.env.QWEN_BASE_URL },
  modelName: process.env.QWEN_MODEL,
  temperature: 0,
}).withStructuredOutput(SentimentSchema)

const text = '这次厦门之旅非常愉快，海鲜很新鲜，但酒店隔音效果一般，总体来说还是值得推荐的。'

const result = await model.invoke([
  new SystemMessage('你是情感分析专家，请分析用户文本的情感。'),
  new HumanMessage(`分析以下文本：${text}`),
])

// result 直接就是 TypeScript 对象，无需 JSON.parse
console.log('情感分析结果:')
console.log('  倾向:', result.sentiment)
console.log('  分数:', result.score)
console.log('  关键词:', result.keywords)
console.log('  摘要:', result.summary)

// TypeScript 类型安全，以下会有类型提示
type SentimentResult = z.infer<typeof SentimentSchema>
const typed: SentimentResult = result
console.log('\n类型验证通过:', typed.sentiment === 'positive' || typed.sentiment === 'negative' || typed.sentiment === 'neutral')
```

**验收**：`result` 直接是 TypeScript 对象，访问字段有 IDE 自动补全，不需要 `JSON.parse`。

**对比旧写法**：如果不用 `withStructuredOutput`，你会在 prompt 里写"请以 JSON 格式返回"，然后 `JSON.parse(response.content)`，经常因为模型多说一个字而崩溃。

## 第二周：记忆 · 上下文 · Runtime

### Day 08 — 会话记忆（checkpointer + thread_id）

**今天学什么**：`MemorySaver`（checkpointer）的作用，以及 `thread_id` 如何隔离不同用户的对话历史。

**做什么**：创建一个带记忆的 Agent，连续提问，让它记住之前说的内容，然后换一个 `thread_id` 验证隔离效果。

```typescript
// day-08.ts
import 'dotenv/config'
import { createReactAgent } from '@langchain/langgraph/prebuilt'
import { MemorySaver } from '@langchain/langgraph'
import { ChatOpenAI } from '@langchain/openai'
import { HumanMessage } from '@langchain/core/messages'

const model = new ChatOpenAI({
  apiKey: process.env.QWEN_API_KEY!,
  configuration: { baseURL: process.env.QWEN_BASE_URL },
  modelName: process.env.QWEN_MODEL,
})

// checkpointer：保存每个 thread 的对话历史到内存
const checkpointer = new MemorySaver()

const agent = createReactAgent({
  llm: model,
  tools: [],
  checkpointer,  // 挂载 checkpointer
})

// 辅助函数：发送消息并打印最后一条回复
async function chat(threadId: string, message: string) {
  const result = await agent.invoke(
    { messages: [new HumanMessage(message)] },
    { configurable: { thread_id: threadId } }  // 关键：指定 thread
  )
  const lastMsg = result.messages.at(-1)!
  console.log(`[${threadId}] 用户: ${message}`)
  console.log(`[${threadId}] 助手: ${String(lastMsg.content)}\n`)
}

// 用户 A 的对话
await chat('user_A', '我叫小明，我在厦门大学读书。')
await chat('user_A', '你还记得我叫什么名字吗？')

// 用户 B 的对话（不应该知道用户 A 的信息）
await chat('user_B', '你知道我叫什么吗？')
```

**验收**：
- 用户 A 第二条问题能得到"小明"的回答
- 用户 B 的回答表示不知道，因为是独立的 `thread_id`

### Day 09 — Runtime Context（安全传递私有变量）

**今天学什么**：`config.context` 的用途——把不应该让大模型"看见"的数据（如 token、userId、db凭证）安全注入给工具。

**做什么**：创建一个"查询成绩"工具，`userId` 不从用户输入获取，而是从 runtime context 中安全注入。

```typescript
// day-09.ts
import 'dotenv/config'
import { createReactAgent } from '@langchain/langgraph/prebuilt'
import { ChatOpenAI } from '@langchain/openai'
import { HumanMessage } from '@langchain/core/messages'
import { tool } from '@langchain/core/tools'
import { z } from 'zod'
import type { RunnableConfig } from '@langchain/core/runnables'

// 模拟数据库
const gradeDB: Record<string, Record<string, number>> = {
  'user_001': { '高数': 92, '英语': 88, '数据结构': 95 },
  'user_002': { '高数': 78, '英语': 85, '数据结构': 70 },
}

// 工具通过 config 获取 userId，而不是从用户消息中读取
// 防止用户伪造其他人的 userId 来查询别人成绩
const queryGradeTool = tool(
  async (input, config: RunnableConfig) => {
    // ✅ 从 runtime context 安全获取（用户看不到，模型也看不到）
    const userId = (config as { context?: { userId?: string } }).context?.userId
    if (!userId) return '错误：未提供用户身份'

    const grades = gradeDB[userId]
    if (!grades) return `用户 ${userId} 暂无成绩记录`

    if (input.subject) {
      const score = grades[input.subject]
      return score != null ? `${input.subject}：${score}分` : `未找到 ${input.subject} 的成绩`
    }

    return Object.entries(grades).map(([k, v]) => `${k}: ${v}分`).join('，')
  },
  {
    name: 'query_grade',
    description: '查询当前登录用户的课程成绩',
    schema: z.object({
      subject: z.string().optional().describe('课程名称，不填则查询所有成绩'),
    }),
  }
)

const agent = createReactAgent({
  llm: new ChatOpenAI({
    apiKey: process.env.QWEN_API_KEY!,
    configuration: { baseURL: process.env.QWEN_BASE_URL },
    modelName: process.env.QWEN_MODEL,
    temperature: 0,
  }),
  tools: [queryGradeTool],
})

// 模拟两个不同用户登录后的查询
for (const userId of ['user_001', 'user_002']) {
  console.log(`\n=== ${userId} 的会话 ===`)
  const result = await agent.invoke(
    { messages: [new HumanMessage('我的高数成绩是多少？')] },
    {
      configurable: { thread_id: userId },
      context: { userId },  // ← 安全注入，不经过大模型
    }
  )
  console.log(String(result.messages.at(-1)!.content))
}
```

**验收**：两个用户查到不同的成绩，且用户无法通过修改问题来查询别人的成绩。

### Day 10 — 消息裁剪 Middleware（trimMessages）

**今天学什么**：长对话会超出模型 context 窗口。`trimMessages` middleware 可以自动裁掉旧消息，保留最近 N 条。

**做什么**：模拟一个长对话，观察 trimMessages 前后的消息数量变化。

```typescript
// day-10.ts
import 'dotenv/config'
import { createReactAgent } from '@langchain/langgraph/prebuilt'
import { MemorySaver } from '@langchain/langgraph'
import { ChatOpenAI } from '@langchain/openai'
import { HumanMessage } from '@langchain/core/messages'
import { trimMessages } from '@langchain/core/messages'

const model = new ChatOpenAI({
  apiKey: process.env.QWEN_API_KEY!,
  configuration: { baseURL: process.env.QWEN_BASE_URL },
  modelName: process.env.QWEN_MODEL,
})

// trimMessages 作为 state_modifier 传入
// 每次调用前自动裁剪：只保留最近 6 条消息，且保留 system message
const trimmer = trimMessages({
  maxTokens: 500,           // 超过500 token 就裁剪
  tokenCounter: (msgs) => msgs.reduce((n, m) => n + String(m.content).length / 2, 0), // 简化估算
  strategy: 'last',         // 保留最新的
  includeSystem: true,      // 始终保留 system message
  allowPartial: false,
})

const agent = createReactAgent({
  llm: model,
  tools: [],
  checkpointer: new MemorySaver(),
  messageModifier: trimmer,  // ← 挂载裁剪器
})

// 发送多轮对话，超出 token 限制后会自动裁剪
```

### Day 11 — Tool 访问 State（config.state）

**今天学什么**：工具除了接收用户输入的参数，还可以通过 `config` 读取当前对话的完整 State（包括历史消息），用于做基于上下文的判断。

**做什么**：创建一个"智能提醒"工具，它能感知到这是第几轮对话，并根据对话轮数给出不同的提示。

```typescript
// day-11.ts
import 'dotenv/config'
import { createReactAgent } from '@langchain/langgraph/prebuilt'
import { MemorySaver } from '@langchain/langgraph'
import { ChatOpenAI } from '@langchain/openai'
import { HumanMessage } from '@langchain/core/messages'
import { tool } from '@langchain/core/tools'
import { z } from 'zod'
import type { RunnableConfig } from '@langchain/core/runnables'

const smartReminderTool = tool(
  async (input, config: RunnableConfig) => {
    // 读取当前 State 中的消息历史
    const state = (config as { store?: unknown; messages?: unknown[] } & RunnableConfig)
    // 通过 config.configurable 获取 thread_id
    const threadId = config.configurable?.thread_id as string ?? 'unknown'

    // 简单统计当前轮数（每两条消息 = 一轮对话）
    const roundCount = Math.floor(((state as {messages?: unknown[]}).messages?.length ?? 0) / 2)

    let reminder = `提醒内容：${input.content}`
    if (roundCount < 2) {
      reminder += '\n（这是您第一次使用，欢迎！）'
    } else if (roundCount > 5) {
      reminder += `\n（您已经进行了 ${roundCount} 轮对话，是个老用户了！）`
    }

    return reminder
  },
  {
    name: 'set_reminder',
    description: '设置一条提醒，系统会根据使用情况给出个性化提示',
    schema: z.object({
      content: z.string().describe('提醒内容'),
    }),
  }
)

const agent = createReactAgent({
  llm: new ChatOpenAI({
    apiKey: process.env.QWEN_API_KEY!,
    configuration: { baseURL: process.env.QWEN_BASE_URL },
    modelName: process.env.QWEN_MODEL,
    temperature: 0,
  }),
  tools: [smartReminderTool],
  checkpointer: new MemorySaver(),
})

const threadId = 'user-state-demo'
const messages = [
  '帮我设置一个提醒：明天开会',
  '好的，再设置一个：下午3点喝水',
  '再来一个：记得提交作业',
  '最后一个：晚上锻炼30分钟',
]

for (const msg of messages) {
  const result = await agent.invoke(
    { messages: [new HumanMessage(msg)] },
    { configurable: { thread_id: threadId } }
  )
  console.log(`\n用户: ${msg}`)
  console.log(`助手: ${String(result.messages.at(-1)!.content)}`)
}
```

**验收**：随着对话轮数增加，工具返回的内容中包含不同的个性化提示。

### Day 13 — RAG 基础（向量检索 + 知识注入）

**今天学什么**：RAG 的核心三步：**切块 → 向量化 → 检索**。理解为什么不能直接把整个文档塞给模型。

**做什么**：把一段长文本（如公司介绍）向量化存储，然后让 Agent 通过检索工具精准回答问题。

```typescript
// day-13.ts（核心片段，完整版自行补充 dotenv/import）
import { ChatOpenAI } from '@langchain/openai'
import { OpenAIEmbeddings } from '@langchain/openai'
import { MemoryVectorStore } from 'langchain/vectorstores/memory'
import { createReactAgent } from '@langchain/langgraph/prebuilt'
import { tool } from '@langchain/core/tools'
import { z } from 'zod'

// 模拟知识库文档（实际可从文件读取）
const documents = [
  '厦门大学信息学院成立于1997年，目前有计算机科学与技术、软件工程等专业。',
  '信息学院每年招收本科生约200人，研究生约150人，博士生约50人。',
  '学院位于厦门大学翔安校区，拥有现代化实验室和科研设施。',
  '保研推免比例约为20%，综合成绩由学业成绩和加分两部分组成。',
  '加分项目包括学科竞赛、科研成果、社会工作等，最高加分不超过12分。',
]

// 1. 向量化存储
const embeddings = new OpenAIEmbeddings({
  apiKey: process.env.QWEN_API_KEY!,
  configuration: { baseURL: process.env.QWEN_BASE_URL },
  modelName: 'text-embedding-v3',
})

const vectorStore = await MemoryVectorStore.fromTexts(documents, {}, embeddings)

// 2. 封装成 Tool
const searchKnowledgeTool = tool(
  async (input) => {
    const results = await vectorStore.similaritySearch(input.query, 3)
    if (results.length === 0) return '知识库中没有相关信息。'
    return results.map((r, i) => `[${i + 1}] ${r.pageContent}`).join('\n')
  },
  {
    name: 'search_knowledge',
    description: '在学院知识库中检索与问题相关的信息',
    schema: z.object({ query: z.string().describe('检索关键词') }),
  }
)

// 3. 接入 Agent
const agent = createReactAgent({
  llm: new ChatOpenAI({ apiKey: process.env.QWEN_API_KEY!, ... }),
  tools: [searchKnowledgeTool],
})

const result = await agent.invoke({
  messages: [{ role: 'user', content: '保研的加分项目有哪些？最高能加多少分？' }],
})
console.log(String(result.messages.at(-1)!.content))
```

**验收**：模型能从知识库中检索到相关片段并给出准确回答，而不是靠自己"记忆"。

### Day 14 — Guardrails（PII 检测 + 输入过滤）

**今天学什么**：如何在 Agent 前后加防护层，过滤敏感信息或不合规输入。

**做什么**：创建一个中间件，检测用户输入中的手机号/身份证号（PII），在送给模型前自动脱敏。

```typescript
// day-14.ts（核心思路）

// 自定义 messageModifier 作为 Guardrail
function piiGuardrail(messages: BaseMessage[]): BaseMessage[] {
  return messages.map(msg => {
    if (msg._getType() !== 'human') return msg
    let content = String(msg.content)
    // 脱敏手机号
    content = content.replace(/1[3-9]\d{9}/g, '1xx-xxxx-xxxx')
    // 脱敏身份证号
    content = content.replace(/\d{17}[\dXx]/g, '****身份证****')
    return new HumanMessage(content)
  })
}

const agent = createReactAgent({
  llm: model,
  tools: [],
  messageModifier: piiGuardrail,  // 每次调用前自动过滤
})
```

**验收**：发送包含手机号的消息，在 Agent 的推理日志中看到脱敏后的版本。

### Day 15 — LangGraph StateGraph 初探

**今天学什么**：`StateGraph` 是比 `createReactAgent` 更底层的工作流引擎。理解"节点（Node）"和"边（Edge）"的概念。

**做什么**：用 StateGraph 手动实现一个两节点的工作流：先"分析意图"，再根据意图"分流"到不同的处理节点。

```typescript
// day-15.ts
import 'dotenv/config'
import { StateGraph, END, START } from '@langchain/langgraph'
import { ChatOpenAI } from '@langchain/openai'
import { HumanMessage } from '@langchain/core/messages'
import { z } from 'zod'

// 定义 State 结构
interface MyState {
  userInput: string
  intent: 'weather' | 'greeting' | 'unknown'
  response: string
}

const model = new ChatOpenAI({
  apiKey: process.env.QWEN_API_KEY!,
  configuration: { baseURL: process.env.QWEN_BASE_URL },
  modelName: process.env.QWEN_MODEL,
  temperature: 0,
}).withStructuredOutput(z.object({
  intent: z.enum(['weather', 'greeting', 'unknown']),
}))

// 节点1：分析意图
async function analyzeIntentNode(state: MyState): Promise<Partial<MyState>> {
  const result = await model.invoke([
    new HumanMessage(`判断以下句子的意图：${state.userInput}`)
  ])
  console.log('  → 意图分析结果:', result.intent)
  return { intent: result.intent }
}

// 节点2a：处理天气意图
async function handleWeatherNode(state: MyState): Promise<Partial<MyState>> {
  return { response: `收到天气查询请求："${state.userInput}"，当前天气晴，25°C` }
}

// 节点2b：处理问候意图
async function handleGreetingNode(state: MyState): Promise<Partial<MyState>> {
  return { response: `你好！有什么可以帮助你的吗？` }
}

// 节点2c：处理未知意图
async function handleUnknownNode(state: MyState): Promise<Partial<MyState>> {
  return { response: `抱歉，我目前只能回答天气和问候相关的问题。` }
}

// 构建图
const graph = new StateGraph<MyState>({
  channels: {
    userInput: { default: () => '', reducer: (_, x) => x },
    intent: { default: () => 'unknown' as const, reducer: (_, x) => x },
    response: { default: () => '', reducer: (_, x) => x },
  }
})
  .addNode('analyze', analyzeIntentNode)
  .addNode('weather', handleWeatherNode)
  .addNode('greeting', handleGreetingNode)
  .addNode('unknown', handleUnknownNode)
  .addEdge(START, 'analyze')
  .addConditionalEdges('analyze', (state) => state.intent)  // 根据意图分流
  .addEdge('weather', END)
  .addEdge('greeting', END)
  .addEdge('unknown', END)

const app = graph.compile()

for (const input of ['厦门今天天气怎么样？', '你好', '帮我写一首诗']) {
  console.log(`\n输入: ${input}`)
  const result = await app.invoke({ userInput: input })
  console.log(`输出: ${result.response}`)
}
```

**验收**：三个不同输入被路由到不同的处理节点，打印结果能看到意图分流过程。

## 常见问题备查

**Q：DashScope 和 OpenAI 的 API 有什么区别？**

LangChain 使用 OpenAI 兼容接口，只需设置 `configuration.baseURL` 和正确的 `apiKey`，其余调用方式完全相同。Embedding 模型名用 `text-embedding-v3`，Chat 模型名用 `qwen3-max`。

**Q：`createReactAgent` 和笔记里的 `createAgent` 是同一个东西吗？**

`createReactAgent` 来自 `@langchain/langgraph/prebuilt`，是目前稳定版中推荐使用的 API。笔记中的 `createAgent` 来自 `langchain` 包（新统一 API），功能一致但包来源不同，优先用 `createReactAgent`。

**Q：工具执行出错了，Agent 会怎么办？**

工具抛出异常时，LangChain 会自动把错误信息包成 `ToolMessage` 送回模型，模型可以看到错误并决定重试或换一个工具，不会直接崩溃。

**Q：什么时候用 `withStructuredOutput`，什么时候用 `tool()`？**

- `withStructuredOutput`：你想要模型的**最终答案**是一个固定格式的对象（如情感分析结果）
- `tool()`：你想让模型在**推理过程中**调用某个能力（如查天气、查数据库），可以多次调用

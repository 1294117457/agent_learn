## 一、修改内容汇总

### 1. ID-AIDemo（Agent）

| 文件 | 操作 | 说明 |
|------|------|------|
| `src/routes/analyze.ts` | **简化** | 移除 mysql.ts 导入及 DB fallback 逻辑，templates 完全由 idbackend 传入；清理误导性注释 |

**核心变化**：
- 原来：`templates` 为空时尝试从 MySQL 查询（独立测试模式）
- 现在：`templates` 为空则直接以空模板列表继续分析（允许 AI 返回空推荐）
- 生产环境不受影响，idbackend 调用时始终传入完整模板列表

### 2. idbackend（后端）

| 文件 | 状态 |
|------|------|
| `service/businessService/AIAgentService.java` | ✅ 已完成（OkHttp，含异常兜底） |
| `controller/businessController/ChatController.java` | ✅ 已完成（agent 不可达返回 503） |
| `controller/businessController/KnowledgeController.java` | ✅ 已完成（管理员权限） |
| `controller/businessController/AIAnalyzeController.java` | ✅ 已完成（DB 查模板 → 传 agent） |
| `resources/application.yml` | ✅ 已配置 `ai-agent.base-url=http://localhost:3001` |
| `pom.xml` | ✅ OkHttp 4.12.0，无千帆 SDK |

**降级兜底机制**（agent 不可达时）：
- `AIAgentService` 所有方法捕获 `Exception`，返回 `{"code":500,"msg":"Agent 不可达","data":null}`
- `ChatController` 检测到 `reply == null` 返回 `HTTP 503`
- `AIAnalyzeController` 检测到 `code != 200` 返回友好错误消息
- 前端非 AI 功能（加分申请、用户管理等）完全不受影响

## 三、测试用例

### T1 — 非 AI 功能正常运行（agent 不可达）

**前提**：不启动 ID-AIDemo
**测试步骤**：

1. 登录学生端 → 进入"加分申请"页面
   **预期**：页面正常加载，模板卡片显示正常，AI 分析入口卡片显示

2. 点击任一模板卡片 → 打开申请弹窗 → 选属性/输入值 → 上传证明材料 → 提交
   **预期**：提交成功，与原有流程完全一致

3. 点击悬浮球 AI 助手 → 发送消息
   **预期**：显示提示"AI 服务暂时不可用，请稍后重试"（HTTP 503）

4. 点击"AI 智能分析" → 上传 PDF → 点击"开始 AI 分析"
   **预期**：显示友好错误提示"AI 服务暂时不可用，请手动申请"

**结论**：agent 不可达时原有功能 ✅ 完全正常，AI 功能 ✅ 降级提示

### T3 — AI 证书分析（完整流程）

**前提**：启动三端 + 准备一份 PDF 证书（如竞赛获奖证书、论文录用通知）
**数据库要求**：`score_templates` 表有激活的模板数据

**步骤1 — 分析**：
1. 学生端 → 加分申请页 → 点击"AI 智能分析"
2. 点击文件区域 → 选择 PDF 证书 → 点击"开始 AI 分析"

4. 点击合适的推荐项

**步骤3 — 确认并提交**：
5. 检查 AI 预填数据（templateName, applyScore, remark）
6. 可手动修改备注
7. 上传证明材料（该 PDF 文件）
8. 点击"提交申请"
   **预期**：提交成功，在"申请历史"中可查看

**测试接口**：
- `POST /api/ai/analyze/certificate` → idbackend → `POST localhost:3001/analyze/certificate`

## 四、关键逻辑验证点

### 降级机制验证
```
场景                     预期行为
─────────────────────────────────────────────────────────
agent 未启动             非 AI 功能完全正常；AI 功能返回友好错误
agent 已启动但模型不可用   分析接口返回 500，前端提示"AI 分析失败，请稍后重试"
agent 已启动且正常        全功能可用
```

### 前后端分离验证（template 传递）
```
调用链: 前端 → POST /api/ai/analyze/certificate (无需传 templates)
         → AIAnalyzeController: 从 DB 查 score_templates
         → AIAgentService: 序列化为 JSON 传给 Agent
         → Agent: 解析 templates，调用 LLM 分析
         → 返回 suggestions[]
```

Agent 不再直连数据库，职责清晰：
- **数据层**：idbackend 负责，从 MySQL 读模板
- **AI 分析层**：Agent 负责，接受 templates 参数进行 LLM 分析

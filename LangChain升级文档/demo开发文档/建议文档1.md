# AI Agent 建议文档1

> 基于现有保研加分系统，将 ID-AIDemo 建设为独立 AI 微服务，对接 idbackend 实现四项 AI 功能。

---

## 一、现状梳理

### 已有基础
| 位置 | 内容 |
|------|------|
| `idbackend` | `ChatController` + `QianFanService`，提供 `/api/chat/send` 和 `/api/chat/clear` |
| `idfrontend` | `ai-agent/index.vue` 悬浮聊天窗（抽成龟）+ `apiAIagent.ts` |
| `idfrontend-admin` | 同上，已有相同聊天组件 |
| `ID-AIDemo` | LangChain + Qwen3 + Express + multer + pdf-parse + better-sqlite3，仅有一个简单对话示例 |

### 问题
- 现有 QianFan 对话无 RAG（检索增强），不了解系统业务知识，回答质量低
- 没有知识库管理能力
- 没有证明材料 PDF 分析能力

---

## 二、整体架构

```
idfrontend / idfrontend-admin
        |
        ▼
   idbackend (Java, :8080)
        |  通过 HTTP 内部调用（转发 AI 请求）
        ▼
   ID-AIDemo (Node.js, :3000)   ←→   Qwen3 API (DashScope)
        |
    SQLite (知识库 chunks + 向量)
```

**核心原则**：所有 AI 请求都先经过 idbackend 做鉴权，再内部转发给 ID-AIDemo。前端无需感知 ID-AIDemo 的存在，保持现有 API 路径不变。

---

## 三、四项功能设计

### 功能1：知识库管理（老师后台）

**新增页面**：`idfrontend-admin/src/views/knowledge/index.vue`

老师可上传以下格式文档：
- PDF（政策文件、评分标准）
- Word（.docx）
- Excel（.xlsx，加分条目表格）
- Markdown（手动整理的规则文档）

**流程**：
```
老师上传文件
  → POST /api/knowledge/upload（idbackend）
    → 转发给 ID-AIDemo POST /internal/knowledge/upload
      → 解析文本 → 切片（chunk） → 存入 SQLite（知识块表）
        → 返回成功
```

**ID-AIDemo 新增**：
- `src/routes/knowledge.ts`：上传、列表、删除接口
- `src/services/docParser.ts`：PDF/Word/Excel/MD 解析
- `src/services/vectorStore.ts`：切片存储与相似度检索（基于 SQLite + 余弦相似度）
- 需要新增依赖：`mammoth`（Word）、`xlsx`（Excel）、`@langchain/textsplitters`

**SQLite 表结构**：
```sql
CREATE TABLE knowledge_chunks (
  id INTEGER PRIMARY KEY,
  source_file TEXT,        -- 原始文件名
  content TEXT,            -- 文本块内容
  embedding TEXT,          -- JSON 数组，Qwen embedding 向量
  created_at INTEGER
);
```

---

### 功能2：RAG 对话（学生+老师均可用）

**升级现有聊天**：将 idbackend 的 `QianFanService` 替换为调用 ID-AIDemo 的 RAG 接口（或 idbackend 直接转发）。

**ID-AIDemo RAG 流程**：
```
用户提问
  → 对问题文本调用 Qwen embedding 得到向量
    → 在 SQLite knowledge_chunks 表做余弦相似度检索，取 top-5 相关块
      → 拼装 System Prompt："你是保研加分助手，以下是相关知识库内容：[chunks]"
        → 调用 Qwen3 生成回答
```

**idbackend 改动**：
- `ChatController` 的 `/api/chat/send` 改为转发给 ID-AIDemo，而非调用 QianFanService
- 或者新增 `/api/chat/rag/send`，旧接口保留兼容

**多轮对话**：在 ID-AIDemo 中按 userId 维护 `conversationHistory`（内存 Map 或 SQLite 表），每次对话追加历史消息。

---

### 功能3：证明材料分析 → 推荐加分条目

**学生操作**：在加分申请页上传 PDF 证明材料，AI 自动识别内容，推荐可以申请的加分项和分数。

**流程**：
```
学生上传 PDF
  → POST /api/score/analyze（idbackend）
    → 转发给 ID-AIDemo POST /internal/analyze/pdf
      → pdf-parse 提取文本
        → 从 knowledge_chunks 检索相关加分规则
          → Qwen3：结合规则和证明内容，输出 JSON 格式推荐结果
            → idbackend 返回给前端
```

**返回格式示例**：
```json
{
  "suggestions": [
    {
      "templateName": "学科竞赛-挑战杯",
      "scoreLevel": "国家级二等奖",
      "estimatedScore": 8,
      "reason": "证明材料显示：挑战杯全国大学生创业计划竞赛国家二等奖"
    }
  ],
  "rawText": "..."
}
```

**前端**（`idfrontend/src/views/score/index.vue`）：
- 在申请页新增「上传证明材料让AI分析」按钮
- 展示 AI 推荐卡片，学生点击一键填入申请表单

---

### 功能4：AI 辅助生成申请

**在功能3基础上扩展**：AI 不仅推荐条目，还帮助填写完整的申请请求体。

**流程**：
```
(在功能3分析结果的基础上)
  → 学生选择某条推荐 → 点击「AI 辅助生成申请」
    → POST /api/score/generate-application（idbackend）
      → 转发给 ID-AIDemo POST /internal/generate/application
        → Qwen3：根据证明材料内容 + 选中的加分模板，生成申请表单数据
          → 返回结构化申请数据（与 POST /api/score/submit 接口字段对齐）
            → 前端自动填入申请表单，学生确认后提交
```

## 六、新增依赖（ID-AIDemo）

```bash
pnpm add mammoth xlsx @langchain/textsplitters
# mammoth: Word 文档解析
# xlsx: Excel 文档解析
```

## 八、实施顺序建议

1. **先搭 ID-AIDemo 服务骨架**：Express 多路由 + SQLite 初始化 + .env 配置
2. **实现知识库上传与存储**（功能1）：docParser + vectorStore
3. **实现 RAG 对话**（功能2）：升级现有聊天，验证知识库效果
4. **实现 PDF 分析**（功能3）：analyze 接口
5. **实现申请生成**（功能4）：generate-application 接口
6. **idbackend 添加代理接口**：按上表新增，做鉴权转发
7. **前端联调**：知识库管理页 + 学生上传分析功能

## 九、注意事项

- **Qwen Embedding**：调用 `text-embedding-v3` 模型（DashScope 同一 BaseURL），每次上传文档时批量生成向量存入 SQLite
- **知识库冷启动**：建议先手动整理一份保研政策 MD 文档作为初始知识，确保对话质量
- **ID-AIDemo 端口**：建议 `:3001`（避免与 idbackend `:8080` 和前端 `:5173` 冲突）
- **安全**：ID-AIDemo 只监听 localhost，idbackend 通过 `http://localhost:3001/internal/...` 内部调用，不对外暴露
- **Excel 支持**：xlsx 库解析 xls/xlsx 后转为文本（表头+行），再切片，列表型数据效果好

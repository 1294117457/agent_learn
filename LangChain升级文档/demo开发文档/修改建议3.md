## 二、新增需求：agent 连接数据库

### 问题

当前 `/analyze/certificate` 的模板列表（`templates` 字段）由 idbackend 调用时传入。
这导致：**脱离 idbackend 独立测试时，必须手工构造完整的 templates JSON**，不方便。

### 建议方案：可选的 MySQL 直连（用于独立测试）

**原则**：生产环境不变（idbackend 传 templates），仅在 `.env` 配置了 MySQL 时才启用直连。

**实现方式**：

#### 1. 新增 `.env` 配置项

```env
# 可选：agent 直连后端数据库（仅用于独立测试，生产环境不配置）
MYSQL_HOST=localhost
MYSQL_PORT=3306
MYSQL_USER=root
MYSQL_PASSWORD=your_password
MYSQL_DATABASE=iddata
```

#### 2. 新建 `src/db/mysql.ts`

```typescript
import mysql2 from 'mysql2/promise'

let pool: mysql2.Pool | null = null

export function getMysqlPool(): mysql2.Pool | null {
  if (!process.env.MYSQL_HOST) return null
  if (!pool) {
    pool = mysql2.createPool({
      host:     process.env.MYSQL_HOST,
      port:     Number(process.env.MYSQL_PORT ?? 3306),
      user:     process.env.MYSQL_USER,
      password: process.env.MYSQL_PASSWORD,
      database: process.env.MYSQL_DATABASE,
    })
  }
  return pool
}

/** 从 idbackend 数据库读取所有激活的加分模板（含规则） */
export async function fetchTemplatesFromDb(): Promise<ScoreTemplate[]> {
  const pool = getMysqlPool()
  if (!pool) return []

  const [templates] = await pool.query<mysql2.RowDataPacket[]>(
    `SELECT id, template_name, template_type, score_type, template_max_score
     FROM score_templates WHERE is_active = 1`
  )

  const result: ScoreTemplate[] = []
  for (const t of templates) {
    const [rules] = await pool.query<mysql2.RowDataPacket[]>(
      `SELECT id, rule_name, rule_score FROM score_template_rules WHERE template_id = ?`,
      [t.id]
    )
    result.push({
      id: t.id,
      templateName: t.template_name,
      templateType: t.template_type,
      scoreType: t.score_type,
      templateMaxScore: t.template_max_score,
      rules: rules.map(r => ({ id: r.id, ruleName: r.rule_name, ruleScore: r.rule_score }))
    })
  }
  return result
}
```

#### 3. 修改 `routes/analyze.ts` 中的 `/certificate` 接口

**改动仅一处**：当 `templates` 字段为空时，尝试从 MySQL 查询：

```typescript
// 原来
let templates: ScoreTemplate[] = []
if (templatesRaw) {
  templates = JSON.parse(templatesRaw)
}

// 改为
let templates: ScoreTemplate[] = []
if (templatesRaw) {
  templates = JSON.parse(templatesRaw)
} else {
  // 未传模板 → 尝试从 DB 获取（仅在配置了 MYSQL_HOST 时生效）
  templates = await fetchTemplatesFromDb()
  if (templates.length > 0) {
    console.log(`[analyze] 使用 DB 模板 ${templates.length} 条（独立测试模式）`)
  }
}
```

**结果**：
- **生产环境**（idbackend 传 templates）：行为不变
- **独立测试**（配置 MYSQL_HOST）：不传 templates，agent 自己查 DB
- **无 DB 无 templates**：返回建议列表为空（而非报错），可接受

## 六、关于 agent 是否需要长期连接数据库

**建议：生产环境不连接，仅测试时连接。**

理由：
- 生产链路是 idbackend 传 templates → agent 分析，职责清晰
- 如果 agent 直连 DB，相当于两套服务都持有 DB 连接，增加维护成本
- 若 DB 表结构变更，需同时更新 agent 和 backend 两处查询逻辑

MySQL 直连仅在 `.env` 配置了 `MYSQL_HOST` 时启用，生产环境不配置该项，行为自动回退到"由 idbackend 传模板"。

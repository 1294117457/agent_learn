## 一、替换现有千帆 AI（直接替换，不再降级）

### 当前状态
`ChatController` → `QianFanService`（百度千帆 AppBuilder，需 Redis 维护会话）

### 替换方案
直接将 `ChatController` 改为调用 `AIAgentService`（调 ID-AIDemo），删除 QianFan 相关代码。

**涉及文件**：

| 文件 | 操作 |
|------|------|
| `ChatController.java` | 移除 `QianFanService` 注入，改为 `AIAgentService` |
| `QianFanService.java` | 删除 |
| `QianFanConfig.java` | 删除 |
| `application.yml` | 移除 `qianfan.*` 配置，新增 `ai-agent.*` 配置 |
| `pom.xml` | 移除百度 AppBuilder SDK 依赖 |

会话管理由 ID-AIDemo 内部的 SQLite `conversations` 表接管，**Redis 中 `chat:conversation:*` 键也无需再维护**。

修改后 `ChatController` 核心逻辑：
```java
@PostMapping("/send")
public ResultVo chat(@RequestBody ChatRequestDTO request, HttpServletRequest httpRequest) {
    if (request.getMessage() == null || request.getMessage().trim().isEmpty()) {
        return ResultVo.error(400, "消息不能为空");
    }
    Integer userIdInt = (Integer) httpRequest.getAttribute("userId");
    String sessionId = userIdInt != null ? "user_" + userIdInt : "anonymous";
    String reply = aiAgentService.chat(request.getMessage(), sessionId);
    if (reply == null) {
        return ResultVo.error(503, "AI服务暂时不可用，请稍后重试");
    }
    return ResultVo.success("成功", reply);
}
```

### 功能3：上传证明材料 PDF → 推荐加分条目

**❌ Agent 端未实现**，需新增 `POST /analyze/certificate`。

**设计方案**：

```
学生上传 PDF（前端）
  → POST /api/ai/analyze/certificate（idbackend，新增）
    → 读取已有 ScoreTemplate 列表（从 DB 获取）
    → 上传 PDF + 模板列表 → POST /analyze/certificate（agent）
      → 解析 PDF 文本
      → RAG 检索加分政策
      → Qwen3 分析，输出匹配的模板+规则+分数+理由
    → 返回推荐列表给前端
```

**Agent 新接口请求体**（multipart）：
```
file: PDF 文件
templates: JSON 字符串（idbackend 传入的加分模板列表）
```

**返回结构**：
```json
{
  "code": 200,
  "data": {
    "certificateText": "提取的 PDF 原文...",
    "suggestions": [
      {
        "templateId": 1,
        "templateName": "学科竞赛加分",
        "ruleId": 3,
        "ruleName": "国家级二等奖",
        "estimatedScore": 4.5,
        "reason": "证明材料中发现：全国大学生xxx竞赛国家二等奖"
      }
    ]
  }
}
```

### 功能4：上传证明材料 PDF → 生成申请请求

**❌ Agent 端未实现**，需新增 `POST /analyze/generate`。

**设计方案**：

```
学生在功能3结果中选择某条推荐（前端）
  → POST /api/ai/analyze/generate（idbackend，新增）
    → 从 JWT 获取学生信息（studentId, studentName, major 等）
    → 取完整模板详情
    → POST /analyze/generate（agent）
      → Qwen3：根据证书文本 + 选定规则，生成申请备注和分数
    → 返回预填的 ScoreApplicationIDTO 给前端
```

**Agent 新接口请求体**：
```json
{
  "certificateText": "PDF 提取的原文",
  "selectedTemplateId": 1,
  "selectedRuleId": 3,
  "template": { "templateName": "...", "templateType": "CONDITION", ... }
}
```

**返回结构**（与 `ScoreApplicationIDTO` 字段对齐）：
```json
{
  "code": 200,
  "data": {
    "templateName": "学科竞赛加分",
    "templateType": "CONDITION",
    "scoreType": 0,
    "applyScore": 4.5,
    "ruleId": 3,
    "remark": "全国大学生xxx竞赛国家二等奖（AI辅助识别）"
  }
}
```

前端拿到后直接填入申请表单，学生确认后调 `/api/application/submit` 正式提交。

## 四、交互关系图（最终）

```
前端（Vue）
  │
  ├─ POST /api/chat/send        → ChatController → AIAgentService → /chat/send（agent）
  ├─ GET  /api/knowledge/list   → KnowledgeController             → /knowledge/list（agent）
  ├─ POST /api/knowledge/upload → KnowledgeController             → /knowledge/upload（agent）
  ├─ DEL  /api/knowledge/:f     → KnowledgeController             → /knowledge/:f（agent）
  ├─ POST /api/ai/analyze/certificate → AIAnalyzeController → 取模板 → /analyze/certificate（agent）
  └─ POST /api/ai/analyze/generate   → AIAnalyzeController → 取学生信息 → /analyze/generate（agent）
```

Agent 不主动调用 idbackend，模板数据由 idbackend 在调用时一并传入。

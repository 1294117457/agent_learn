### 一、 核心 AI 功能实现

核心的 AI 能力主要围绕 **LangChain** 框架来调度阿里云 Qwen 大模型。

#### 1. RAG 对话 (基于知识库的问答)

- **实现原理**：接收用户提问 -> Embedding 化 -> 在本地向量库进行相似度检索 -> 取出最相关的 Top-5 知识块 -> 与历史对话和系统 Prompt 拼接 -> 请求大模型并以流式（SSE）返回。
- **涉及技术**：`LangChain` 框架的 ChatOpenAI 模块兼容对接（封装 Qwen）、`Server-Sent Events (SSE)` 流式输出。
- **对应代码**：
  - RAG 逻辑核心：[src/services/ragChain.ts](vscode-file://vscode-app/c:/Program Files/Microsoft VS Code/resources/app/out/vs/code/electron-browser/workbench/workbench.html)
  - 对话接口层 (SSE)：[src/routes/chat.ts](vscode-file://vscode-app/c:/Program Files/Microsoft VS Code/resources/app/out/vs/code/electron-browser/workbench/workbench.html)

#### 2. 证书/证明材料分析

- **实现原理**：针对上传的 PDF 证书等材料，先进行文本提取，通过 RAG 检索最匹配的加分政策，然后再把后端提供的可用“加分模板”、证书原文一并发送给大模型，利用低温度值 (`temperature: 0.1`) 让大模型按要求输出严格格式的 JSON （推荐的加分项）和一段文字备注。
- **涉及技术**：大模型的格式化抽取与推理、LangChain 的提示词模板。
- **对应代码**：
  - 分析生成逻辑：[src/services/analyzeChain.ts](vscode-file://vscode-app/c:/Program Files/Microsoft VS Code/resources/app/out/vs/code/electron-browser/workbench/workbench.html)
  - 系统接入接口：[src/routes/analyze.ts](vscode-file://vscode-app/c:/Program Files/Microsoft VS Code/resources/app/out/vs/code/electron-browser/workbench/workbench.html)

### 二、 辅助支持与文件功能

#### 1. 文档读取与解析

- **实现原理**：将不同格式的知识库文档、凭证解析为纯文本，以便后续提供给大模型或者进行向量切割。
- **涉及技术**：
  - `.pdf`：`pdf-parse`（仅限文字型PDF解析）
  - `.docx`/`.doc`：`mammoth`（Word转普通文本）
  - `.xlsx`/`.xls`：`xlsx`（将表格按行/表头转文本）
  - 文档上传接收：`multer`（Express 文件上传中间件）
- **对应代码**：
  - 文档解析工具：[src/services/docParser.ts](vscode-file://vscode-app/c:/Program Files/Microsoft VS Code/resources/app/out/vs/code/electron-browser/workbench/workbench.html)
  - 上传与管理接口：[src/routes/knowledge.ts](vscode-file://vscode-app/c:/Program Files/Microsoft VS Code/resources/app/out/vs/code/electron-browser/workbench/workbench.html)

#### 2. 向量生成与检索 (轻量级知识库)

- **实现原理**：读取文档文本，按照约500字做滑动窗口分块，调用大模型接口（`text-embedding-v3`）成批量生成向量（JSON 浮点数组），并存放到 SQLite 单表中。当需要检索时，将全表记录载入内存执行**余弦相似度计算 (Cosine Similarity)** 进行排序获取 Top-K。
- **涉及技术**：
  - 持久化存储：`better-sqlite3`（轻量同步数据库驱动）
  - Embedding API 调用封装
- **对应代码**：
  - 向量生成调用：[src/services/embeddings.ts](vscode-file://vscode-app/c:/Program Files/Microsoft VS Code/resources/app/out/vs/code/electron-browser/workbench/workbench.html)
  - 本地检索与余弦计算：[src/services/vectorStore.ts](vscode-file://vscode-app/c:/Program Files/Microsoft VS Code/resources/app/out/vs/code/electron-browser/workbench/workbench.html)
  - 冷启动与自动入库（扫描本地文件）：[src/seed/seedKnowledge.ts](vscode-file://vscode-app/c:/Program Files/Microsoft VS Code/resources/app/out/vs/code/electron-browser/workbench/workbench.html)

#### 3. 动态配置

- **实现原理**：使用 SQLite 存储 `system_role`、`api_key`、`chat_model` 等 AI 参数，并通过内存缓存策略实现调用大模型工厂函数时的“热加载/热更新”，保障老师后台修改后立即生效。

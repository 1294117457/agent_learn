## 一、当前功能实现总结

### 1.1 技术栈

| 层次       | 技术                                           |
| :--------- | :--------------------------------------------- |
| 运行时     | Node.js + TypeScript (ESM)                     |
| HTTP 框架  | Express 5                                      |
| AI 模型    | 阿里云 DashScope（Qwen3-max，兼容 OpenAI API） |
| LangChain  | @langchain/core + @langchain/openai            |
| 本地存储   | SQLite（better-sqlite3）                       |
| 可选数据库 | MySQL（连接 idbackend 的 iddata 库）           |
| 文档解析   | mammoth（docx）/ pdf-parse / xlsx              |

------

### 1.2 模块结构与功能

src/

├── index.ts              # Express 启动入口，挂载路由

├── db/

│   ├── init.ts           # SQLite 初始化：3张表 + 默认 ai_config

└── routes/               # Express 路由层

## 三、升级优先级建议

| 优先级 | 升级项                                              | 收益                | 难度 |
| :----- | :-------------------------------------------------- | :------------------ | :--- |
| 🔴 立即 | 启用 INTERNAL_TOKEN 鉴权中间件                      | 安全性              | 低   |
| 🔴 立即 | 接入 fetchTemplatesFromDb()（代码已写好）           | 减少 idbackend 耦合 | 低   |
| 🟡 近期 | createAgent + Tools 重构 analyzeChain               | 自主纠错能力        | 中   |
| 🟡 近期 | checkpointer + summarizationMiddleware 替换手写历史 | 对话质量            | 中   |
| 🟢 中期 | LangGraph StateGraph 证书分析工作流                 | 复杂分析能力        | 高   |
| 🟢 中期 | 异步分析 + Webhook 回调机制                         | 超时问题            | 中   |
| 🔵 长期 | HITL 人工审核节点                                   | 可靠性              | 高   |
| 🔵 长期 | MCP 标准化对外能力                                  | 可扩展性            | 高   |

------

总结：当前 agent 的 LangChain 基础层（LCEL + 结构化输出 + ChatPromptTemplate）已经比较扎实，短期升级的重点应放在后端交互的安全性和解耦（鉴权 + 模板自动拉取）以及用 createAgent + Tools 替换线性分析链，让分析流程具备自主推理和纠错能力。LangGraph StateGraph 和 HITL 则是中长期的架构演进方向。
